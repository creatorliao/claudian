# 调查报告：Claude Agent、Cursor Agent、OpenAI Codex 的官方 SDK / 程序化接入方式

**调查目的**：确认三款「编码智能体」是否都有官方提供的 **SDK 或等价程序化接口**，并基于**官方网站/官方文档**整理开发者如何接入、大致能达成什么效果。

**信息来源与日期**：以各产品官方文档为主（2026-04-21 检索与摘录）。下文链接均为官方域名；若文档改版，请以官网为准。

---

## 执行摘要

| 产品 | 是否有「官方库级 SDK」 | 其他官方程序化方式 | 对开发者的大致定位 |
|------|------------------------|--------------------|--------------------|
| **Anthropic Claude（Agent / Claude Code 能力）** | **有** — TypeScript：`@anthropic-ai/claude-agent-sdk`；Python：`claude-agent-sdk` | 与 CLI 共用配置与工作流；另可经 Bedrock / Vertex / Azure 等路由 | 把 **Claude Code 同款 agent 循环、内置工具、上下文管理** 嵌进自有应用 |
| **Cursor** | **有（Beta）** — npm：`@cursor/february` | **Cursor Agent CLI**（`agent`，支持 `-p` / `--output-format`）；**Cloud Agents REST API**（Beta）；**ACP**（`agent acp`，stdio JSON-RPC，偏高级集成） | 用 **同一套 TS API** 驱动本机或云端 Agent；或用 CLI/HTTP 做自动化与集成 |
| **OpenAI Codex** | **有** — TypeScript：`@openai/codex-sdk`；Python 侧有 **实验性**、面向 app-server 的封装（见官网说明） | **`codex app-server`**（JSON-RPC / stdio，富客户端首选）；**`codex mcp-server`**（MCP 工具形态）；**Codex Exec**（轻量脚本/CI，官网文章提及） | 在应用内 **线程化控制 Codex**；或深度对接 **与 VS Code 扩展同级的事件流（审批、历史、流式 item）** |

**结论（直接回答「是不是都有对应 SDK」）**：

- **三款在官方文档层面都可以找到「库级」或「明确维护的程序化入口」**。
- 形态不同：**Claude** 与 **Codex** 的 npm SDK 名称清晰；**Cursor** 的 TypeScript SDK 包名为 **`@cursor/february`**，文档标注为 **Beta**，且与 **CLI、Cloud API** 并列存在。
- 若仅讨论「像 Anthropic 那样单一 `npm install xxx-sdk` 且 GA」：**Cursor 的 SDK 仍处于 Beta**；**Codex 的 Python SDK 在官网明确写为 experimental**。

## 1. Anthropic — Claude Agent SDK（Claude Code 同款能力库）

### 1.1 官方信息要点

- **文档入口（示例）**：[Agent SDK overview / TypeScript](https://docs.anthropic.com/en/docs/agent-sdk/typescript)（Anthropic / Claude Code 文档体系；另有 `docs.claude.com` 等镜像路径）。
- **定位**：原「Claude Code SDK」已更名为 **Claude Agent SDK**；用于构建可自主读文件、跑命令、改代码等的生产级 Agent，与 Claude Code **同款工具、agent 循环与上下文管理**。
- **安装**：
  - TypeScript：`npm install @anthropic-ai/claude-agent-sdk`
  - Python：`pip install claude-agent-sdk`
- **认证**：常见为环境变量 `ANTHROPIC_API_KEY`；文档亦提及 Bedrock / Vertex / Azure 等第三方提供方配置方式。
- **使用形态**：文档示例为 `query({ prompt, options })` 异步迭代消息；内置 Read / Edit / Bash 等工具，**无需自研完整工具执行层即可开跑**（与「仅 Messages API + 自管工具」不同）。

### 1.2 开发者接入后「大概能达成什么效果」

- 在 **自有产品、内部平台、IDE 插件** 中嵌入与 Claude Code **同源的 agent 行为**（文件、终端、编辑、权限/MCP 等由 SDK 选项与配置支撑）。
- 适合：**深度定制 UI/流程**，但仍希望 **复用官方 agent 运行时**，而不是自己拼 LLM + 工具协议。

### 1.3 参考链接

- https://docs.anthropic.com/en/docs/agent-sdk/typescript  
- https://docs.anthropic.com/en/api/agent-sdk/typescript

## 2. Cursor — TypeScript SDK（Beta）、CLI、Cloud API、ACP

### 2.1 官方信息要点

**（1）TypeScript SDK（Beta）**

- **文档**：[Cursor APIs Overview](https://cursor.com/docs/api) 中列出 **TypeScript SDK | Beta**，详述见 [TypeScript SDK](https://cursor.com/docs/api/sdk/typescript.md)。
- **包名**：`npm install @cursor/february`
- **能力概述**：
  - 用 **`Agent.create()`** 统一封装 **Local**（本机工作区）与 **Cloud**（Cursor 托管或自托管 worker）运行时。
  - **`agent.send()`** 得到 **`Run`**，可 **`for await` 流式消费** `assistant` / `thinking` / `tool_call` / `status` 等事件（文档称 **`SDKMessage` 在各运行时形态一致**）。
  - 支持 **MCP 内联配置**、**子 Agent（Sub-agents）**、恢复会话 **`Agent.resume`**、列出 agents/runs/messages 等。
- **认证**：`CURSOR_API_KEY` 或传入 `apiKey`；文档说明可与 Cloud Agents 仪表板密钥等配合。

**（2）Cursor Agent CLI（非 npm SDK，但是官方一等集成方式）**

- **文档**：[CLI Overview](https://cursor.com/docs/cli/overview)、[Using Agent in CLI](https://cursor.com/docs/cli/using)、[Parameters](https://cursor.com/docs/cli/reference/parameters)。
- **典型用途**：交互式终端；脚本/CI 使用 **`-p` / `--print`** 非交互模式，并可配合 **`--output-format json` 或 `text`** 做结构化输出。
- **ACP**：文档说明可通过 **`agent acp`** 以 **stdio + JSON-RPC** 做自定义客户端（高级场景）。

**（3）Cloud Agents API**

- **文档**：[Cursor APIs Overview](https://cursor.com/docs/api) 中 **Cloud Agents API | Beta (All Plans)**，与 HTTP 端点配合使用。

### 2.2 开发者接入后「大概能达成什么效果」

- **用 `@cursor/february`**：在 Node/TS 服务或工具链里 **统一驱动本机与云端** Cursor Agent，获得 **与文档一致的流式事件模型**，便于做内部 IDE、编排平台、自动化评审流水线。
- **用 CLI**：快速接入 Shell/GitHub Actions；需 **细粒度流式协议** 时可结合文档中的 **output-format / stream-json** 等（与具体 CLI 版本一致，以 Cursor CLI 文档为准）。
- **用 Cloud API**：从服务端创建/管理云端 Agent 任务，适合 **长任务、PR、脱离本机仓库** 的场景。

### 2.3 参考链接

- https://cursor.com/docs/api  
- https://cursor.com/docs/api/sdk/typescript.md  
- https://cursor.com/docs/cli/overview  
- https://cursor.com/docs/cli/using

## 3. OpenAI Codex — npm SDK、App Server、MCP、Exec

### 3.1 官方信息要点

**（1）Codex SDK（TypeScript）**

- **文档**：[SDK – Codex | OpenAI Developers](https://developers.openai.com/codex/sdk/)
- **包名**：`npm install @openai/codex-sdk`
- **要求**：服务端使用；**Node.js 18+**
- **使用形态（文档示例）**：`new Codex()` → `startThread()` → `thread.run(prompt)`；可在同一线程多次 `run`，或 `resumeThread(threadId)` 恢复。
- **效果描述（官网原文意图）**：用于 CI/CD、构建与 Codex 交互的自有 Agent、内部工具、把 Codex **嵌进自有应用**。

**（2）Python SDK（实验性）**

- 同一页面说明：Python 库为 **experimental**，通过 **本地 Codex app-server 的 JSON-RPC** 控制；安装与仓库路径见官网（需 **开源 Codex 仓库** 上下文及 Python 3.10+）。

**（3）Codex App Server（富集成 / 与 IDE 扩展同级能力）**

- **文档**：[App Server – Codex | OpenAI Developers](https://developers.openai.com/codex/app-server/)
- **协议**：**JSON-RPC 2.0**；传输含 **stdio（JSONL）** 等；需 **`initialize` → `initialized`** 再 `thread/start` 等（详见文档）。
- **工具链**：文档提供 **`codex app-server generate-ts`** / **`generate-json-schema`** 从当前 Codex 版本生成模式，便于自写客户端。
- **与 SDK 文档的取舍**：App Server 页面写明——若主要是 **自动化/CI**，可优先考虑 **Codex SDK**；若要在产品内做 **认证、会话历史、审批、流式 agent 事件** 等 **深度集成**，用 **app-server**。

**（4）与 OpenAI Agents SDK / MCP**

- **文档**：[Use Codex with the Agents SDK](https://developers.openai.com/codex/guides/agents-sdk) — 通过 **`codex mcp-server`** 把 Codex 作为 MCP 工具接入 Agents SDK 编排。

**（5）官方博客对「集成路径」的归纳**

- [Unlocking the Codex harness（OpenAI）](https://openai.com/index/unlocking-the-codex-harness) 说明 **App Server** 是面向客户端的主要接口，并对比 **MCP 方式**（编排友好、能力面相对窄）、**Codex SDK**（TS 库、早于 App Server、语言/覆盖面相对有限）等。

### 3.2 开发者接入后「大概能达成什么效果」

- **`@openai/codex-sdk`**：最快在 **Node 服务** 内 **起线程、多轮对话、恢复线程**，适合 **流水线与内部工具**。
- **`codex app-server`**：自建客户端（任意语言）实现与 **Codex VS Code 扩展同级** 的交互深度（事件流、审批、账户等，以文档为准）。
- **`codex mcp-server`**：在已有 **MCP / Agents SDK** 体系里把 Codex 当 **可调用工具**。

- https://developers.openai.com/codex/guides/agents-sdk

## 4. 给开发者的选型提示（非官方，仅归纳）

1. **要「官方维护的库 + 少写协议」**  
   - Claude：`@anthropic-ai/claude-agent-sdk`  
   - Cursor：`@cursor/february`（注意 **Beta**）  
   - Codex：`@openai/codex-sdk`  

2. **要「最全 UI 级事件与管控面」**  
   - Codex 侧文档明确倾向 **`codex app-server`**。  
   - Cursor 侧 **`@cursor/february` 的 stream 事件** + Cloud API 组合需按业务拆分。  

3. **要「Shell/CI 最快落地」**  
   - Cursor：**`agent -p`** 与非交互参数。  
   - Codex：文档中的 **Exec** 路径 + SDK 的「控制 Codex as part of CI/CD」描述。  

4. **要「和自家 Agent 框架拼在一起」**  
   - Claude：Agent SDK 与 MCP/钩子体系。  
   - Codex：**MCP + OpenAI Agents SDK** 指南。  
   - Cursor：**MCP 内联**（TS SDK 文档）或 CLI 规则/MCP 配置。
