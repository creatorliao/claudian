# C01 需求清单 · 提供商设置页（Claude 等）精简与分层展示

**项目目录**：`docs/01-Projects/R20260429-02-提供商设置页精简/`  
**创建日期**：2026-04-29  
**状态**：需求记录（待方案确认与实施指令）  
**关联**：与 `docs/01-Projects/R20260429-01-精简界面/` 中「通用」页简易模式同一产品方向延续；本需求聚焦**各提供商子页签**（以 **Claude** 为主；Codex / Cursor 是否同期对齐见下文「范围」）。

---

## 1. 背景与问题

当前在设置中切换到 **Claude**（及其他提供商）页签时，单页同时呈现：`Setup`（CLI 路径）、`Safety`、`Models`（含长上下文开关与自定义模型）、斜杠命令与隐藏命令、子智能体、MCP、Plugins、环境与实验功能等。对**初次使用、以对话为主**的用户而言，信息量大、术语多，易产生畏惧或不知从何下手。

已在「通用」页通过「显示更多选项」做了第一层减负；提供商页仍需**面向小白用户**再做一轮：**默认少而够用**，进阶能力折叠或后置。

---

## 2. 目标

在**不删除能力**的前提下：

1. 提供商页默认只暴露与「日常对话、命令/技能/规则类能力」强相关的区块；
2. **路径、模型微调、环境、实验项**等归入「更多」或置于页尾，避免占据首屏；
3. 默认值与心智对齐：**接受编辑**、**加载全局配置**；
4. **命令行路径（依赖自动检测）**在简易模式下置于**最后**（或归入高级区块末尾），满足「多数人不改也能用」。

## 4. 功能需求

### 4.1 总体交互（与「通用」页一致的方向）

| 编号 | 需求描述 |
|------|----------|
| R1 | 提供商页增加 **「显示更多选项」**（或共用同一文案体系），**默认关闭**。关闭时仅展示 **§4.2 简易清单** 中的内容。 |
| R2 | 开关打开后，展示当前版本中**其余全部**提供商专属配置（与现有行为等价，顺序可按 §5 重组）。 |
| R3 | 开关状态**持久化**（写入插件设置），重启 Obsidian 后保持。 |

### 4.2 简易模式（默认）下须保留并优先展示的内容

站在小白用户角度，**默认应能看见、且靠前**的区块如下（用语可与现有 i18n 对齐后定稿）：

| 优先级 | 用户表述 | 对应实现侧（当前 Claude 页结构） | 说明 |
|--------|----------|-------------------------------------|------|
| P1 | **命令、技能、规则**等与智能体相关的组件 | **Slash Commands**（含 Vault 级命令/技能列表、`SlashCommandSettings`）、必要时含 **Subagents**、**Plugins（Marketplace 插件）** 中与「命令/技能/规则」一致的区块 | 用户明确要求「这些智能体的组件应该被展示出来」；具体边界（例如 MCP 是否算「组件」）见 §6 待决。 |
| P2 | **安全 / 编辑策略**简单可理解 | **Safety** 中与「接受编辑」相关的选项（如 `safeMode`：**默认「接受编辑」**） | 与需求「默认就是接受编辑」一致；若当前默认已是 `acceptEdits`，则简易模式下可仅展示简短说明或下拉默认态，避免堆砌术语。 |
| P3 | **加载全局配置** | **`loadUserSettings`**（加载用户全局 Claude 设置）：**默认开启** | 与需求「默认加载全局配置」一致；文案面向用户解释「使用你在其他环境的全局习惯」（实施时再精炼）。 |

**不应**在简易模式首屏占据大块面积的项（归入「显示更多选项」或整体后置）包括但不限于：

- **CLI 自定义路径**（含按主机名存储的路径）：需求 **§4.3 第 2 点**明确要求——**默认自动检测的路径配置放在最后**（即简易模式下不出现或仅在展开/页尾出现）。
- **Models** 内：长上下文（Opus/Sonnet 1M）开关、**自定义模型 ID 列表**等偏进阶内容。
- **MCP Servers**、**Environment（提供商专属环境变量）**、**Experimental**（Chrome / Bang-Bash 等），除非产品确认归入「组件」范畴（当前文档默认**隐藏**，见 §6）。
- **Hidden slash commands**（隐藏命令列表）：可与「命令」区分——简易模式可只保留「正面清单」管理，隐藏列表放进高级（待方案确认）。

### 4.3 顺序与默认值（明确摘录原始诉求）

| 编号 | 原始诉求 | 需求化表述 |
|------|----------|------------|
| O1 | 复杂配置像前面一样藏起来 | 与「通用」页相同的 **渐进披露**：默认折叠高级区块，行为见 R1–R2。 |
| O2 | 命令行工具，默认自动检测的放在最后 | **CLI 路径（Setup）**在简易模式中**不置于顶部**；整体置于 **页签后部**或仅在「显示更多选项」内展示；留空即自动检测的路径语义保持不变。 |
| O3 | 默认就是接受编辑的 | **`safeMode`（或等价设置）默认值为「接受编辑」**（`acceptEdits`）；简易模式下可见且默认选中该项；若已有默认，实施时核对 `DEFAULT_CLAUDE_PROVIDER_SETTINGS`，避免回归。 |
| O4 | 命令、技能、规则等组件展示；其他复杂配置隐藏；默认加载全局配置 | **Slash Commands / Skills / Plugins（规则相关 UI）**在简易模式下**保留展示**；**`loadUserSettings` 默认开启**并在简易模式下可见或可一句话说明。 |

## 5. 简易模式下的推荐信息架构（草案 · 供方案设计对照）

以下为 **Claude** 页「默认关闭显示更多选项」时的**由上到下**顺序草案（实施时可微调）：

1. 「显示更多选项」开关及简短说明（与通用页文案风格一致）。
2. **Safety（精简）**：突出「接受编辑」；可选一行说明「加载全局配置」与 `loadUserSettings` 开关。
3. **Slash Commands / Skills**（及文档确认的「规则」所在区块）。
4. **Subagents**（若产品认定属于「智能体组件」——保留）。
5. **Plugins**（若与技能/规则强相关——保留）。
6. （可选）其余极简提示或指向文档的链接。

**「显示更多选项」打开后**：补齐 **Setup（CLI）**（且按需求放在**高级区块的偏后位置**）、完整 **Models**、**Hidden commands**、**MCP**、**Environment**、**Experimental** 等，顺序以实现侧最少惊讶为原则。

---

## 6. 待决问题（方案阶段需闭合）

1. **「规则」**在用户话术中指代的具体 UI：是否等同于 Vault 下某类存储 + **Plugins**，还是包含 **MCP**？若 MCP 对小白仍过重，建议默认归入高级。
2. **Codex / Cursor** 提供商页是否与 Claude **同一套**「显示更多选项」开关语义，还是分提供商记忆——影响设置字段设计。
3. **简易模式下 `loadUserSettings` 的呈现**：独立开关 vs 合并进一句说明 + 高级再展开。

## 7. 非功能需求

| 编号 | 说明 |
|------|------|
| N1 | 现有用户已保存的提供商配置不得因分层展示而被静默清空或改写；仅调整 UI 分组与默认值应用于**新安装或缺失字段回填**的策略需在方案中写明。 |
| N2 | 国际化：新增/调整的标题与描述须写入 `zh-CN.json` / `en.json`；延续「用户可见文案不出现不必要品牌堆砌」的用语原则（与 R20260429-01 文档边界一致时可对齐）。 |
| N3 | 无障碍与一致性：开关命名与「通用」页「显示更多选项」保持一致或明确父子关系，避免两套「高级」含义冲突。 |

## 8. 验收要点（草案）

- [ ] 默认进入 **Claude** 设置页时，「显示更多选项」为关闭，首屏以 §4.2 / §5 为准，**CLI 路径不作为首屏第一项**。
- [ ] 打开「显示更多选项」后，与当前版本相比**能力不减少**（均有入口）。
- [ ] 默认 **接受编辑**、默认 **加载全局配置**（与产品默认值及持久化数据一致）。
- [ ] 命令 / 技能 / 规则相关组件在简易模式下**可见**（范围以方案确认为准）。
