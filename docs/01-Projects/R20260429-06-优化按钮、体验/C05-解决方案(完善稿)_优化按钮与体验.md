## 2. R-01：刷新按钮（最终实现说明）

### 2.1 根因（确认）

- `SlashCommandSettings`：`refreshBtn` 使用 `claudian-sp-header-action-trailing`（`margin-left: auto`），在存在「查看全部」「+」时仍把 **刷新顶到最右**（`src/providers/claude/ui/SlashCommandSettings.ts`）。
- `AgentSettings` / `CodexSubagentSettings` / `CodexSkillSettings`：**先**创建 `refreshBtn`、**后**创建 `addBtn` → 视觉顺序为 **刷新 | +**，与上者相反。

### 2.2 推荐补丁形态（防止二次偏斜）

对每个区块：

1. 调整创建顺序：**先** `addBtn`，**后** `refreshBtn`。
2. 给 `refreshBtn` 增加 class：`claudian-settings-action-btn claudian-sp-header-action-trailing`（与命令预览一致）。

如此即使未来在两者左侧增加文字按钮，刷新仍会靠最右。

**文件**：  
`src/providers/claude/ui/AgentSettings.ts`  
`src/providers/codex/ui/CodexSubagentSettings.ts`  
`src/providers/codex/ui/CodexSkillSettings.ts`  
样式已存在：`src/style/settings/base.css`（`.claudian-sp-header-action-trailing`）。

### 2.3 风险

- **极低**：不涉及网络与状态机，仅 DOM 顺序与 class。

### 3.1 行为锚点（避免再次误读）

## 4. 斜杠命令列表：触发、收起与选中路径（源码级论证）

以下均基于当前 `src/shared/components/SlashCommandDropdown.ts` 与 Tab 事件线。

### 4.1 列表如何出现

- 组件在构造时对 `inputEl` 监听 **`input`**，回调为 `handleInputChange()`（同文件约 L67–L68）。
- `handleInputChange()` 从光标向左扫描，仅在「触发符（默认 `/`）位于行首或空白之后」且「触发符与光标之间无空白」时调用 `showDropdown()`；否则 **`hide()`**（约 L118–L128）。

### 4.2 列表如何收起（全部路径）

| 路径 | 触发条件 | 代码位置 |
|------|----------|----------|
| **A. 选中一项** | 鼠标点击某 `.claudian-slash-item` → `selectItem()`；或键盘 Enter/Tab 且 `filteredItems.length > 0` | `selectItem()` 末尾 **`this.hide()`**（约 L417）；`handleKeydown` 中 Enter/Tab 调 `selectItem()`（约 L148–L154） |
| **B. Esc** | 下拉可见时 | `handleKeydown` → `this.hide()`（约 L156–L158） |
| **C. 输入不再构成合法 slash 片段** | 删除 `/`、在片段中输入空格、光标移出片段等 | `handleInputChange()` 中 `triggerIndex === -1` 或 `searchText` 含空白 → `hide()`（约 L118–L127） |
| **D. 过滤结果为空且搜索串非空** | 用户继续输入使无匹配 | `showDropdown()` 中 `searchText.length > 0 && filteredItems.length === 0` → `hide()`（约 L227–L229） |
| **E. 禁用组件** | `setEnabled(false)` | 直接 `hide()`（约 L71–L74） |
| **F. Bang bash 等** | Tab 侧在 bang 激活时对 dropdown `setEnabled(!isActive)` | `wireTabInputEvents` / `syncBangBashSuppression`（`Tab.ts` 约 L1424–L1429） |

**选中后的关键顺序**（保证收起先于下游回调）：

```403:421:src/shared/components/SlashCommandDropdown.ts
  private selectItem(): void {
    if (this.filteredItems.length === 0) return;

    const selected = this.filteredItems[this.selectedIndex];
    if (!selected) return;

    const text = this.getInputValue();
    const beforeTrigger = text.substring(0, this.triggerStartIndex);
    const afterCursor = text.substring(this.getCursorPosition());
    const replacement = `${selected.insertPrefix}${selected.name} `;

    this.setInputValue(beforeTrigger + replacement + afterCursor);
    this.setCursorPosition(beforeTrigger.length + replacement.length);

    this.hide();
    if (selected.slashCommand) {
      this.callbacks.onSelect(selected.slashCommand);
    }
    this.inputEl.focus();
  }
```

- **`hide()`** 会移除 `visible` 类并将 **`triggerStartIndex` 置为 `-1`**（`hide()` 约 L168–L174），界面层已收起。
- 插入内容以 **`名称 + 尾随空格`** 结束；光标停在空格之后。此后若再次触发 `handleInputChange()`（例如用户继续键入），从光标向左扫描会先遇到 **空白** 并 **中断** 扫描，得不到合法 `triggerIndex`，进入 **`hide()`** 分支（与路径 **C** 一致），**不会**在「已结束的一轮插入」上无故长期挂起列表。

### 4.3 与 `onSelect` / 现网空实现的说明

- Tab 初始化里 `onSelect`、`onHide` 目前为 **空函数**（`Tab.ts` `initializeSlashCommands` 约 L685–L686）；内联编辑器同样为空（`InlineEditModal.ts` 约 L440–L441）。
- **收起完全不依赖** `onSelect`：仅靠 **`selectItem()` 内的 `hide()`** 即可保证列表关闭；空回调不影响收起。
- 将来若在 `onSelect` 内做较重逻辑，仍应保持 **先 `hide()` 再 `onSelect`** 的现有顺序，避免回调异常中断导致状态不一致（当前实现已满足）。

### 4.4 程序化改字数与「误触发再打开」

- 常规浏览器下，对 `<textarea>` **赋值 `value`** **不会**派发 `input` 事件，故 **`selectItem()` 仅改值不会自动再跑一遍 `handleInputChange()`**。
- 若实施工具条时在其它路径 **`dispatchEvent('input')`**：在 `selectItem` 已完成、文本形如 `/commandName ` 且光标在尾部的情况下，向左扫描会先遇空格而 **`triggerIndex === -1`** → `hide()`，**仍不会**错误保持展开。
- **实施约束**：工具条打开列表时，优先 **`handleInputChange()`**（在已正确插入 `/` 并设置选区之后），避免手写一半状态与 `triggerStartIndex` 不一致。

### 4.5 与导航/Escape 的统一

- `NavigationController` 的 `shouldSkipEscapeHandling` 在 **`slashCommandDropdown.isVisible()`** 为真时会拦截，避免 Esc 被其它层吞掉（`Tab.ts` 约 L1407）。
- **`hide()` 后** `isVisible()` 为假，Esc 行为回到导航/取消流；**与现网一致**，工具条触发不需改导航层。

### 4.6 已知体验缺口（非本需求阻断）

- **点击列表外区域** 关闭：当前 `SlashCommandDropdown` **未**实现全局 pointerdown 捕获；用户主要靠 Esc 或消除 `/` 片段关闭。若要做「点外部关闭」，需 **单独任务**（注意与 `@` 提及、模态层 z-index 的竞态）。

## 5. R-03：工具条按钮（完善后的实施规格）

### 5.1 API 建议（类 `SlashCommandDropdown`）

新增 **公开** 方法（命名可二选一，实现等价）：

- `openSlashPickerFromToolbar(): void`

职责：

1. 若 `!this.enabled`，直接 `return`（与 bang bash 等一致）。
2. `inputEl.focus()`。
3. 读取 `selectionStart` / `selectionEnd` 与全文，在**选区折叠为一个插入点**的前提下：
   - 若该点前已存在合法 slash 片段（与 `handleInputChange` 判定一致），**不再插入**第二个 `/`，必要时把光标移动到「紧跟在合法 `/` 之后、搜索子串末尾」以便列表与键盘一致。
   - 否则在 **行首** 或 **紧跟空白** 处插入 `/`（若插入点位于词中间，实施上应 **先** 插入分隔空格再 `/`，或 **禁止** 在词中打开并仅 focus——以产品取舍为准，**推荐**与手动输入 `/` 相同规则：仅在行首或空白后）。
4. 调用 **`this.handleInputChange()`** 刷新列表。
5. 若使用 `fixed` 布局（内联编辑），可依赖现有 `render()` 内 **`positionFixed()`**（约 L358–L360），无需重复实现定位。

**禁止**：在 Tab 外再维护一份 `ProviderCommandEntry` 列表或重复 `showDropdown` 的过滤排序。

### 5.2 挂载点（侧栏）

- `createInputToolbar`（`InputToolbar.ts`）当前返回模型/权限/MCP 等；**建议**在工厂内 **新增** `SlashCommandPickerButton` 小组件，或在该行末尾追加一个 **与现有 ribbon 同风格** 的 `div` + `setIcon`（如 `slash` / `terminal` 等，需实机选）。
- `Tab.ts` 的 `initializeTabUI` 中 **`initializeSlashCommands` 先于 `initializeInputToolbar`**（约 L949–L964），因此创建工具条时 `tab.ui.slashCommandDropdown` **已存在**，可在 `initializeInputToolbar` 内直接绑定按钮，无需额外延迟。

### 5.3 内联编辑器（`InlineEditModal`）

- 已有独立 `SlashCommandDropdown`（`fixed: true`，挂 `document.body`）。
- 若验收要求「内联一并」：在 `inputWrap` 内 **输入框左侧或右侧** 增加与侧栏 **相同 aria 文案** 的按钮，点击同一 `openSlashPickerFromToolbar()`。
- 若分期交付：可在 C01 验收中明确 **Phase 1 仅侧栏**，内联 Phase 2，避免范围膨胀未沟通。

### 5.4 国际化

- 新增键示例：`chat.ribbon.openSlashCommands`（中：打开命令与技能列表），仅用于 **`aria-label`**，**勿**再加 `title`（与 `ClaudianView` 头部注释一致，避免双重 tooltip）。
- 更新 `src/i18n/types.ts`、`en.json`、`zh-CN.json`。

### 5.5 回归用例（工具条专项）

- 侧栏：空输入点按钮 → 列表出现 → Enter 选一项 → **列表消失**，正文为 `/name `。
- 侧栏：已有 `hello `，光标在末尾点按钮 → **插入** `/` 于合法位置 → 列表与手动输入一致。
- Bang bash 激活时：按钮无响应或列表不打开（与 `setEnabled(false)` 一致）。
- 内联（若实施）：`fixed` 下拉位置仍贴着输入框。
