## 1. 现象与用户期望差异

当前已在 Claude **命令与技能**配置页实现分区预览、「查看全部」分项弹窗、行上的「编辑 / **文件夹（打开资源管理器）** / 删除（本库）」等。

**本轮反馈**：用户对「行内主操作按钮」的期望不是「打开文件夹」，而是 **删除磁盘上的命令文件或技能子目录**，并与「查看全部」弹窗内列表 **语义一致**；且 **不希望在多条路径上出现多套删除入口**。

与此不一致的现状摘要：

| 区域 | 当前行为 | 用户期望 |
|------|-----------|-----------|
| 配置列表与「查看全部」列表每行行尾 | **文件夹图标**：调用「在文件管理器中显示」对应路径 | **删除图标**：删除该项对应的磁盘实体（命令为文件、技能为子目录），**删除前确认** |
| 同一行的多余控件 | 除文件夹外另有垃圾桶（本库可点 / 本机共用灰色）或并排多个意图相近按钮 | **仅保留唯一入口**：与服务删除语义对齐的主按钮（删除）；不在别处再放同类入口 |

另有产品与 **[C05](C05-解决方案(完善稿)_分离技能和命令.md)** 的差异：

- C05 / 现行代码曾对 **`slashFileProvenance === 'user-home'`** 条目禁用插件内删除（`ClaudeCommandCatalog`：`fromUserHome` 则 `isDeletable: false`），以避免误删用户主目录与其它 Claude Code 环境共用文件。
- **本轮需求**：行内不再区分「本库 vs 本机共用」，删除语义一致——实施后须在文案与安全提示上明确 **删除会影响磁盘实体**，对用户主目录条目附加更强风险提示。

---

## 2. 需求条目（来自反馈）

1. **「查看全部」弹框**：列表行上原先文件夹图标改为 **删除**，删除对象为该条目对应的 **命令文件** 或 **技能子目录**；执行前 **友好确认对话框**。
2. **技能删除粒度**：技能删除视为删除对应磁盘目录树（或当前实现的等价语义）；命令删除视为删除对应文件；均需确认。
3. **配置页主列表**：与「查看全部」弹框 **同样的行为**——同一图标语义（删除）、同样的确认流程；**不**按来源拆分两套图标语义。
4. **单一入口**：除上述配置列表（含预览区与查看全部列表）外，**其它界面不应再提供**独立的「删除命令/技能」入口（需在实施后做一次全局检索核对）。
