## 1. 现状摘要（代码核对）

- **单一容器**：`SlashCommandSettings` 使用常量 `SLASH_COMMANDS_PREVIEW_LIMIT = 5`，对 `commands` **混合**截取预览。
- **全文弹窗**：`SlashCommandsFullListModal` 将 `entries` **单层循环**渲染，无命令/技能分区。
- **删除**：`renderCommandRow` 已在 `cmd.isDeletable` 时渲染删除按钮并调用 `deleteCommand` → `catalog.deleteVaultEntry`。**用户主目录**条目 `isDeletable === false`（见 `ClaudeCommandCatalog.slashCommandToEntry`）。
- **缓存**：删除成功后已有 `reloadCommands()` + `render()` + `notifyVaultSlashCommandsMutated()`；若需与「来源切换」完全一致，可在删除成功后视情况调用 `catalog.refresh()`（与 `ClaudeSettingsTab` 下拉变更路径对齐），避免 SDK/probe 与文件列表短暂不一致。

## 2. 信息架构方案

### 2.1 顶层结构（建议）

在 `renderSlashSkillsSection` 保持顺序：

1. 小节总标题：可改为「自定义命令与技能」或保留「命令与技能」（需 i18n），说明段一句不变或微调。
2. **来源**：保留现有「命令与技能的来源」`Setting` 一行（全局生效）。
3. **命令**：新 `Setting.setHeading()` 或使用与插件列表一致的 **分组标题样式**（`claudian-sp-section-heading` 等若已有），下方嵌入 **仅含命令** 的 `SlashCommandSettings` 子区域或同一类实例传参 `variant: 'commands-only'`。
4. **技能**：同上，仅含技能。

实现上推荐 **复用单个类**，通过构造函数参数 `mode: 'commands' | 'skills' | 'all'` 过滤 `listVaultEntries()` 结果，避免复制两份逻辑。

### 2.2 「系统」与「配置」

建议在每条卡片或分区脚注用**一致术语**，而非再拆第三套顶层 Tab：

| 用户可见标签 | 含义 |
|--------------|------|
| **本库（可编辑）** | `slashFileProvenance === 'vault'`，可保存、删除（删除前确认）。 |
| **本机共用（只读）** | `user-home`，仅打开文件夹 / 预览，**不出删除**。 |

若将来设置页需要列出 SDK 条目：归入「运行时（只读）」分组，`isEditable/isDeletable` 均为 false。

## 3. 预览：各 3 条

- 将 `SLASH_COMMANDS_PREVIEW_LIMIT` 替换为 **每类** `PREVIEW_LIMIT_COMMANDS = 3`、`PREVIEW_LIMIT_SKILLS = 3`。
- 数据：`commandsCmd = all.filter(e => !isSkillEntry(e))`，`commandsSkill = all.filter(isSkillEntry)`，各自 `slice(0, 3)`。
- **头部操作区**（预览标签、「查看全部」、`+`、刷新）：
  - **方案 A（推荐）**：来源行下先放**共用工具条**（一次刷新扫描两类），再分两列/两段列表；「查看全部」仍打开**一个** Modal，内部分区。
  - **方案 B**：命令区与技能区各有独立「查看全部（n）」—— 弹窗标题或初始滚动位置对应一类；开发量略大。

初稿推荐 **方案 A**，与「单一 N」计数一致：`N = commandsCmd.length + commandsSkill.length`。

## 4. 「查看全部」弹窗

- 结构与样式：保留 `claudian-sp-modal`，正文滚动区自上而下：
  1. **子标题「命令（n₁）」** + 列表；
  2. **子标题「技能（n₂）」** + 列表。
- 空的一类可显示一行灰色说明：「暂无自定义命令」/「暂无技能」。
- 顶部文件夹快捷按钮行：保持现有四个按钮逻辑（vault commands/skills + home 条件下 home commands/skills）。

---

## 5. 删除按钮与确认对话框

### 5.1 按钮布局

- 对 **本库可删** 条目：`编辑`（若有）→ **`在文件夹中打开`**（`folder-open`）→ **`删除`**（`trash-2`），与现有 `claudian-settings-delete-btn` 一致。
- 若截图中仅见文件夹图标：多为 **只读本机条目**（无编辑/删除）或窄宽度折叠；本方案明确 **可删条目始终展示删除**，并与打开并列。

### 5.2 确认内容（文案要点）

- 标题：例如「删除自定义命令」/「删除技能」。
- 正文：包含 **`/名称`**、类型（命令文件 vs 技能目录）、以及「将从本库删除且不可恢复」。
- 按钮：**取消** / **删除**（删除为主destructive 样式若主题支持）。

使用 Obsidian `Modal` 或封装的小型 `ConfirmModal`，避免浏览器原生 `confirm()`（可访问性与样式不一致）。

### 5.3 删除后刷新

1. `await catalog.deleteVaultEntry(cmd)`
2. `await catalog.refresh()`（若与切换来源路径对齐需要）
3. `reloadCommands()` / `loadAndRender()`
4. `notifyVaultSlashCommandsMutated()`
5. `Notice` 成功提示（沿用现有 key）
