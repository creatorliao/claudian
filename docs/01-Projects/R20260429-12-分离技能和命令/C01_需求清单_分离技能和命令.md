## 1. 背景与问题

当前「命令与技能」在配置页中以**单一列表**混合展示：条目通过 `/名称` 与「技能」角标区分类型，但仍共用同一标题、同一预览区与同一「查看全部」列表。存在以下问题：

1. **概念混淆**：**自定义斜杠命令**（`.claude/commands/` 等处的文件型命令）与**技能**（`.claude/skills/` 下的技能包）在 Claude Code / 官方文档中为不同资产类型；用户在配置页难以一眼区分「我现在改的是哪一类」。
2. **「系统」与「配置」语义重叠**：用户心智中，运行时探测条目与**可编辑的配置资产**（本库或本机目录下的文件）是两层概念；界面需避免让用户误以为「列表里每一项都能在本插件里同等删除」。
3. **预览占用纵向空间**：希望改为**命令最多 3 条 + 技能最多 3 条**的并行摘要。
4. **「查看全部」**：应采用 **命令与技能各一套「查看全部（n）」与各自弹窗**，而不是合计数字与混合列表（详见 C05）。
5. **删除流程**：删除前确认；删除后刷新列表与缓存（含 `catalog.refresh()` 与斜杠下拉缓存失效）。

---

## 2. 目标（验收口径）

### 2.1 信息架构：两组独立配置区

- **自定义命令**：单独小节（heading），仅列出 `kind !== skill` 的文件型条目。
- **技能**：单独小节，仅列出技能条目。
- **「命令与技能的来源」**（`slashAssetScope`）：仍为**全局一项**，置于两组小节之前，只改一次。

### 2.2 「系统」与「配置」——展示策略（已定）

- **不使用**字面「系统」「配置」作为顶层分组标题。
- 使用 **`本库` / `本机共用`** 标签区分数据位置；本机共用条目不在插件内删除或保存。
- 设置页列表仍以 **磁盘文件扫描**（`listVaultEntries`）为准；**不包含** SDK 运行时独占条目（说明见 C05）。

### 2.3 预览条数

- **命令**预览最多 **3** 条；**技能**预览最多 **3** 条；各自独立。
- 某一类为空时展示该类空态文案。

### 2.4 「查看全部」— **分开**

- **命令**：「查看全部（n₁）」→ 弹窗仅含命令，`n₁` 为命令总数。
- **技能**：「查看全部（n₂）」→ 弹窗仅含技能，`n₂` 为技能总数。
- 弹窗顶部保留打开四大文件夹按钮（本库命令/技能、本机命令/技能，依来源配置）。

### 2.5 删除与缓存刷新

- **删除前**：确认对话框（区分命令文件 vs 技能文件夹）。
- **删除后**：`deleteVaultEntry` → **`catalog.refresh()`** → 重载列表 → `notifyVaultSlashCommandsMutated()`。
