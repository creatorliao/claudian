## 1. 背景与问题

当前「命令与技能」在配置页中以**单一列表**混合展示：条目通过 `/名称` 与「技能」角标区分类型，但仍共用同一标题、同一预览区与同一「查看全部」列表。存在以下问题：

1. **概念混淆**：**自定义斜杠命令**（`.claude/commands/` 等处的文件型命令）与**技能**（`.claude/skills/` 下的技能包）在 Claude Code / 官方文档中为不同资产类型；用户在配置页难以一眼区分「我现在改的是哪一类」。
2. **「系统」与「配置」语义重叠**：用户心智中，**系统侧**（运行时探测、SDK 内置、或不可在库内直接删改的条目）与**可编辑的配置资产**（本库或本机目录下的文件）是两层概念；当前界面若混在同一列表中，容易误认为均可同等管理。
3. **预览占用纵向空间**：现有产品文档曾约定主界面预览约 **5 条**混合摘要（见 `R20260429-05` 系列）；为节省空间，希望改为**命令最多 3 条 + 技能最多 3 条**的并行摘要。
4. **「查看全部」弹窗**：全文列表仍为混合排序时，命令与技能不易扫读；需在弹窗内**明确分区**。
5. **删除流程不完整**：业务上需要支持删除**本库内**可删的命令文件或技能目录（与现有 `catalog.deleteVaultEntry` 能力对齐）；执行删除前应有**确认对话框**，删除成功后刷新列表并同步清空对话侧斜杠下拉等相关缓存。若当前实现已有删除按钮但缺少确认或入口不明显，需在本需求中一并收敛。

## 2. 目标（验收口径）

### 2.1 信息架构：两组独立配置区

- **命令**：单独一个小节（标题 + 简短说明可选），管理文件型**自定义命令**（非技能类条目）。
- **技能**：单独一个小节，管理**技能**条目（`kind === 'skill'`）。

「命令与技能的来源」（`slashAssetScope`）仍为**两者共用**的一项全局策略：放在两个小节**之上**或作为独立一行保留在区块顶部，避免用户改两次来源。

### 2.2 「系统」与「配置」分拆（概念与展示）

- **配置（可管理）**：本库（Vault）内**可编辑、可删除**的命令文件与技能目录——设置页 CRUD 的主要对象。
- **系统 / 只读**：包括但不限于：**本机用户目录下合并展示**且**不可在插件内改写/删除**的条目（现有逻辑：`user-home` 仅展示、`save`/`delete` 抛错）；以及若后续在设置列表中纳入 **SDK/运行时探测**条目，应归入只读区并标注来源，避免与「配置」混在同一交互层级。

本迭代至少做到：**在 UI 文案与分组上**区分「仅展示 / 只读」与「本库可管理」，不与「命令 / 技能」两道分组冲突（详见解决方案文档中的落地选项）。

### 2.3 预览条数

- 主界面：最多展示 **3 条命令**预览 + **3 条技能**预览（各自独立列表或上下两个子列表）。
- 若某一类为 0 条，对应区域展示空态或精简占位，不占用另一类名额。
- 「查看全部」按钮的总数 **N** 仍为两类合计（或产品确认后为两类分别展示数量——实施方案中二选一并保持一致）。

### 2.4 「查看全部」弹窗

- 弹窗内将 **命令** 与 **技能** 分为两个区块（例如两级标题或分隔带），条目样式延续现有卡片/行，但**分区清晰**，便于扫描。
- 顶部的「打开文件夹」快捷按钮若保留，继续区分 commands/skills、vault/home（与现有一致）。

### 2.5 删除与缓存刷新

- 在「在文件管理器中打开」按钮旁（或同一操作组内）提供 **删除** 入口（与现有 `isDeletable` 一致：**仅本库可删**）。
- **删除前**弹出确认框（标题、正文说明删除对象名称与不可恢复性）。
- 确认后：删除磁盘上对应命令文件或技能文件夹 → `catalog` 与运行时相关缓存按现有管线刷新（含 `commandCatalog.refresh()`、`resetSlashDropdownProviderCachesOnAllTabs` 等与来源切换时对齐的行为）→ 列表重绘。

## 3. 非目标（本迭代不承诺）

- 不改变 Obsidian「通用」与「Claude」顶层 Tab 结构（除非实施方案评审另有拆分）。
- 不要求在本迭代内新增对用户主目录文件的**插件内写入/删除**（维持现有安全边界）。
- 不要求改动对话内 `/ ` 下拉的数据结构协议，仅要求设置页变更后缓存失效行为与现有一致或增强。

---

## 4. 待定项（确认方案时拍板）

1. 「查看全部」右侧数字：**合计 N** 还是 **命令 n₁ + 技能 n₂** 两处展示。
2. 「系统 / 配置」是否需要在标题层级出现字面「系统」「配置」二字，还是仅用「本库（可编辑）」「本机共用（只读）」与角标完成区分。
3. SDK/运行时条目是否出现在设置页列表：当前 `listVaultEntries()` 仅文件扫描；若本迭代仍只展示文件型条目，则「系统」分区可为预留文案或仅用于只读条目的二级分组。
