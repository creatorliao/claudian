# C05 解决方案（完善稿）· 通用设置页「显示更多」布局

**项目目录**：`docs/01-Projects/R20260429-07-单工作空间选项/`  
**关联需求**：`C02_需求清单_通用设置页显示更多布局.md`  
**关联既有**：`C01` / `C03`（工作空间开关与刷新管线）——本改动 **只调整设置 UI 布局与条目可见性**，不改变 `allowWorkspaceSwitch` 默认值与各提供商 `enabled` 默认值。

---

## 1. 方案摘要

在 **`ClaudianSettingTab.renderGeneralTab`** 中调整渲染顺序与条件：

1. **始终先渲染「常用」区块**：`renderGeneralUserAndAgent` → `renderGeneralLanguage` → `renderGeneralMaxTabs`（`simpleLabels` 仍由 `!showMore` 驱动，与改前一致）。
2. **页面底部**渲染「显示更多选项」`Setting` 行，并为该行增加与 Claude 页类似的 **顶部分割样式类**（通用页专用类名），避免与上方常用项视觉粘连。
3. **仅当 `showMoreGeneralOptions === true`**：在容器内 **新建包裹元素**（如 `claudian-settings-general-advanced-block`），于其内 **顺序**渲染：
   - 展开说明 hint（沿用 `settings.moreOptions.hintWhenExpanded`）；
   - **工作空间切换** `Setting`（自顶部展区移入此处）；
   - **`renderProviderToggles`**（`simpleModeHint: false`，因已进入进阶区）；
   - **`renderGeneralAdvancedSections`**（原有「显示 / 对话 / 内容 / …」整块下移至此包裹内）。

这样满足：**Toggle 在最后一段常用项之后**；**所有「显示更多」专属项均在 Toggle 之下连续排列**，不与常用项穿插。

---

## 2. 默认值与数据层

- **无需** 修改 `DEFAULT_CLAUDIAN_SETTINGS`：已为 `allowWorkspaceSwitch: false`；Claude `enabled: true`，Cursor/Codex `enabled: false`，满足「默认只开 Claude」。
- 提供商至少保留一个启用的校验逻辑仍在 `renderProviderToggles` 的 `onChange` 中，**不搬迁**。

---

## 3. 样式

在 `src/style/settings/base.css` 增加：

- **通用页「显示更多」行**：顶部分割线与间距（对齐 `.claudian-settings-claude-more-options` 的意图，类名区分以免误伤 Claude 页）。
- **通用页进阶包裹块**（可选）：与 Claude `.claudian-settings-claude-advanced-block` 类似的留白或左边距（若与现有结构对比后需要再微调）。

---

## 4. 测试与回归

- 手动：切换「显示更多」开/关，确认条目顺序与 §4 验收一致。
- 自动化：运行 `npm run typecheck`、`npm run lint`、`npm run test`；若有针对设置结构的单测，按需增补。
