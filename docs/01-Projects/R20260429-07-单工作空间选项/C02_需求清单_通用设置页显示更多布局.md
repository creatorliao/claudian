# C02 需求清单 · 通用设置页「显示更多」布局与条目归属

**项目目录**：`docs/01-Projects/R20260429-07-单工作空间选项/`  
**创建日期**：2026-04-29  
**关联**：在 `C01` 单工作空间开关能力基础上，优化 **通用** 设置页的 **信息架构**，降低初次上手认知负荷。

---

## 1. 背景

当前通用页将「显示更多选项」置于靠前位置，且「允许切换工作空间」、各聊天提供商（Claude / Cursor / Codex）启用开关与常用项穿插排列；用户希望在 **关闭「显示更多」** 时只看到足够日常使用的精简项，并把 **进阶项集中在开关之下连续展示**，避免扫读时被多条目打断。

---

## 2. 需求条目

| 编号 | 需求 |
|------|------|
| R1 | **「显示更多选项」开关** 固定在通用设置页的 **页面底部**（在精简常用项之后）。 |
| R2 | **Claude / Cursor / Codex** 三个聊天提供商的 **启用开关**，仅在 **「显示更多选项」为开启** 时展示；默认只需 **Claude 启用**（与现有各提供商 `enabled` 默认值一致：仅 Claude 为 `true`）。 |
| R3 | **「允许切换工作空间」**（`allowWorkspaceSwitch`）仅在 **「显示更多选项」为开启** 时展示（与 `C01` 语义兼容：进阶用户才暴露切换入口）。 |
| R4 | 凡 **仅因「显示更多」开启才出现** 的设置项，必须 **全部排在「显示更多」Toggle 之后**，在 Toggle **下方** 成组连续展示，**不与** 精简常用项穿插。 |

---

## 3. 精简区（关闭「显示更多」时仍可见）

至少包含：**称呼 / 助手展示名称**、**界面语言**、**最大标签页数**（与既有「简易模式」文案策略一致）。不含：工作空间切换开关、提供商启用列表、系统提示词等进阶块。

---

## 4. 验收关注点

- 关闭「显示更多」：通用页底部为 Toggle；Toggle 上方仅有常用项；**无** 工作空间切换、**无** 提供商三开关。
- 开启「显示更多」：Toggle 下方依次出现说明 hint（若有）、工作空间切换、提供商区块（含 heading）、其余进阶区块（显示 / 对话 / 内容 / 输入 / 快捷键 / 环境等），**顺序连贯**。
- 新安装或默认设置：**仅 Claude 提供商启用**，Codex/Cursor 默认关闭（与 `DEFAULT_*_PROVIDER_SETTINGS.enabled` 一致）。

---

## 5. 实现提示（追踪用）

- 主要改动文件：`src/features/settings/ClaudianSettings.ts` 中 `renderGeneralTab` 及必要时 `src/style/settings/base.css`。
- 不需新增持久化字段；行为与 `showMoreGeneralOptions`、`allowWorkspaceSwitch`、各 `providerConfigs.*.enabled` 既有模型一致。
