# C03 解决方案（初稿）· 单工作空间选项与工作空间全局刷新管线

**项目目录**：`docs/01-Projects/R20260429-07-单工作空间选项/`  
**关联需求**：`C01_需求清单_单工作空间选项.md`  
**基准代码**：检索与阅读截至 **2026-04-29** 的 Claudian `src/` 实现  
**文档目的**：在综合代码审查基础上，划定「工作空间」相关模块边界、补齐**调用链路**说明，并提出 **Toggle 开关** + **切换时全量刷新** 的技术方案；从用户路径与程序路径双向核对，尽量减少回归面。

---

## 1. 结论摘要

| 维度 | 结论 |
|------|------|
| **数据模型** | 存在两处：① `SharedStorageService` 持久化 **默认工作空间**（Vault 相对路径，存在 `plugin.data.json`，空串表示根）；② 每个 **`TabData.workspace`** 为**绝对路径快照**，`null` 表示语义上的库根。**Ribbon 文案**读的也是 ①，与单个 Tab 的 ② **可能不一致**（见 §3.5）。 |
| **运行时 cwd** | **Claude / Codex** 在会话轮次中已通过 `effectiveCwd`/Tab 快照参与 `cwd`（与库根可调不同）；**Cursor** 当前实现 **固定为 Vault 根路径**，与工作空间快照无关（若产品要求子目录亦为 Cursor workspace，列为独立迭代）。 |
| **斜杠命令 / 技能** | **共享**单一 `ClaudeCommandCatalog`，运行时命令缓存在 **`sdkCommands`**；冷启动且无就绪 runtime 时靠 **`probeRuntimeCommands`**。**当前 probe 固定 `cwd: vaultPath`**（库根），与 Tab 子目录工作空间 **未对齐**。 |
| **关键产品缺口（实施前须知）** | 文件库 **右键「设为工作空间」** 当前仅 **`setWorkspace` + 更新 Ribbon**，**不会**刷新已打开 Tab 的 `workspace` 快照——与需求 **R5「切换后刷新配置」** 不一致，需在方案落地时一并处理。 |

本文 §4 给出 **统一管线**；§5 给出 **Toggle** 与各入口对齐方式；§6 为验收与风险控制。

## 3. 调用链路（从用户动作到运行时）

### 3.1 用户路径 A：右键「设为工作空间」

```txt
Explorer 右键文件夹
  → main.ts「file-menu」onClick
     → SharedStorageService.setWorkspace(vault-relative folder path)
     → updateRibbonTooltipFromStorage()   // Ribbon 展示新默认路径
```

**未到路径**：不改变已有 `TabData.workspace`，不调用 `ensureReady`/不刷新命令目录。

---

### 3.2 用户路径 B：新建聊天 Tab（或启动后首次 `createTab`）

```txt
TabManager.createTab(...)
  → initialWorkspace =
       persistedWorkspace 若为恢复
       else resolveWorkspacePath(await storage.getWorkspace(), vaultPath)
  → TabData.workspace = 绝对路径 | null（非法则回落行为见 resolveWorkspacePath）
```

---

### 3.3 用户路径 C：头部「重置工作空间」（home）/ 命令面板 `reset-workspace`

```txt
ClaudianView reset 按钮 → plugin.resetWorkspaceAndNotify()
  → storage.setWorkspace("")
  → Notice + updateRibbonTooltipFromStorage()
  → 对每个已打开 ClaudianView: resetWorkspaceToVaultRoot()  // tab.workspace := null
```

此处 **覆盖了所有 Tab**，与路径 A **不对称**——产品若希望右键与「重置」等价地「全员生效」，应对齐到 §4。

---

### 3.4 用户路径 D：发送消息 / 拉起运行时（抽象）

```txt
Tab.getEffectiveCwd()  // resolveWorkspacePath(tab.workspace) ?? vaultPath
  → ChatRuntime.ensureReady({ effectiveCwd })
  → 轮次 query(..., queryOptions.effectiveCwd)
  → Claude: buildQueryOptionsContext(..., cwd)
  → Codex: CodexLaunchSpec targetCwd 等
```

---

### 3.5 用户可见「两套默认」简述

| 界面位置 | 反映的数据来源 |
|----------|----------------|
| Ribbon 提示后缀 | **`storage.workspace`**（默认工作空间）。 |
| 聊天标题旁的 `· …` | **当前 Tab 的 `tab.workspace`**。 |

右键设路径后：**Ribbon 变、当前 Tab 可能仍指向旧 cwd**，易造成「我明明设了文件夹，为什么副标题没变」的认知问题；**_toggle 全开后的「切换即刻刷新」**应在此一并收口。

## 4. 推荐方案：**工作空间语义统一 + 单次刷新管线**

### 4.1 命名建议（实现层）

抽取 **`WorkspacePolicy`**（名称可任选，此处表意）：

- **`allowWorkspaceSwitch: boolean`**：与 C01 Toggle 对齐；存入 **`ClaudianSettings`**（与其他 UI 布尔一致），并写入 **`defaultSettings`**、`ClaudianSettingsStorage` **迁移默认值**。
- 关闭时：**强制语义** 「所有 Tab **`workspace = null`**，且 **`storage.workspace = ''`**」——回答 C01 §7 待定：**关闭 = 等价库根且不保留子目录默认**，避免出现隐藏子目录仍为 cwd 的回力镖问题。

可选：若在关闭前有非根 `data.workspace`，**一次性清零**或在首次检测到关闭时静默归位（须有明确日志策略；生产不推荐 `console`）。

---

### 4.2 核心：**`applyWorkspaceSwitch(nextRelative: string)`**（或等价）

**单一入口**，供右键、（若保留）程序化设置调用；**严禁**再在业务散落写 `setWorkspace` 而不走刷新。

**建议步骤顺序**（可依实现微调，但必须可审计）：

1. **校验路径**：仅用 Vault 内、目录存在（与现有 `isPathWithinVault`、`resolveWorkspacePath` 一致）。
2. **持久化默认**：`await storage.setWorkspace(nextRelative)`（根 → `''`）。
3. **同步所有打开 Tab**：对每个 `ClaudianView`：
   - 将每个 `Tab.workspace` 设为 **`resolveWorkspacePath(trimmedRelative, vaultPath)`**，根则为 **`null`**（与 **`resetWorkspaceToVaultRoot`** 语义一致）。
4. **`updateRibbonTooltipFromStorage()`**。
5. **每个 Tab 视图 UI**：`**updateWorkspaceSubtitle()`**（或集中 `broadcast`）。
6. **命令 / 技能目录（Claude）**：
   - 对 **所有 provider 的 catalog**（至少 `ClaudeCommandCatalog`，Codex **若另有独立 dropdown 数据源**并行）：
     - 调用 **`commandCatalog.refresh()`**（已实现：清空 **`sdkCommands` + probe 状态**）。
   - **`slashCommandDropdown.resetSdkSkillsCache()`**（多处已有 hook；工作空间切换应对**所有 Tab**触发，与 **`onConversationLoaded`** 同思路）。
7. **运行时就绪与 cwd**：
   - 对已 **`serviceInitialized`** 的 Tab：**按 **`main.ts` env 批量更新**同源逻辑**类推**：
     - 若会话与 cwd 绑定紧耦合：**`tab.service.resetSession()` + `ensureReady({ effectiveCwd: derived, externalContextPaths: ... })`**；
     - 若能热换 cwd：**`ensureReady({ force: true, effectiveCwd })`**。
   - **流式 Tab**：先 **`cancelStreaming`**（与 env 变更处理一致）。
8. **`Claude`** **`AgentManager` / MCP**（若在用户心智中算作「本地项目配置」）：
   - 当前 **AgentManager 以 vault 根构造**——若产品与 Claude Code CLI 对齐为「子文件夹 `.claude`」，需评估：**仅 **`ensureReady` + SDK cwd`** 不够时**，是否在 cwd 切换时 **`ProviderWorkspaceRegistry` 按需回调**「重载 Agents 列表」（可复用 `refreshAgentMentions`，并核对 **mention 数据源**）。
9. **`Codex`**：**`invalidate` skills cache**、`ensureReady`，保证 **`targetCwd`** 与新 workspace 对齐。
10. **`Cursor`**：若 **`allowWorkspaceSwitch` 仍为 off 或本产品不在本期改 Cursor**：在文档与用户文案中写明 **始终以 Vault 为 workspace-dir**。

---

### 4.3 修正 **`probeRuntimeCommands`**

当前 **`cwd`** 固定为库根。**应改为**：接受 **`effectiveCwd: string`**（通常即当前 Tab 或「单一活动 Tab」的 **`getEffectiveCwd()`**，无 Tab 上下文时 **`vaultPath`**）。

- **注入途径**：`**ClaudeCommandCatalogDeps.probe**` 从闭包升级为 **可传参**或由 **`plugin`** 查询 **当前活动 Tab cwd**——注意 **异步与 UI 线程**：probe 前应确定「以哪个 Tab 为准」——**下拉列表打开时始终以「当前激活 Tab」为准**更符合 §3.5。

这同时满足：**「全局与用户目录合并的技能」切换目录瞬间被重新 probe/列出**——因为 **`refresh()` + 新 cwd probe** 等价于重做「冷启动列举」路径。

---

### 4.4 Toggle **`allowWorkspaceSwitch` 关掉时**

| 位置 | 行为 |
|------|------|
| **`main.ts` `file-menu`** | **不添加**「设为工作空间」菜单项（或使用 `checking`/`if`）；若事件无法取消注册，则用条件包裹 `addItem`。 |
| **`ClaudianView` 头部 home** | **`display:none`** 或不创建该 **`resetWsBtn`**（推荐 **创建后隐藏**，避免 Hover Editor 等问题下 DOM 次序突变）。 |
| **命令 `reset-workspace`** | **`checkCallback`**：关掉时返回 **`false`**（命令面板不可见或无操作）。 |
| **副标题 **`workspaceSubtitleEl`** | 两种策略任选其一：**(a)** 仍可显示 **`·`** 但只有「库根」文案；**(b)** 完全隐藏整条副标题。**需求 R7 侧重「不出现切换按钮」**，副标题若为纯展示根路径，是否与 R7 冲突：**建议关 Toggle 时仍显示「库根」或隐藏副标题**——产品二选一写入 C04。 |

**数据**：执行 **§4.1** 所述强制归 **`null`/`''`**。

## 6. 用户视角与程序视角的核对表

### 6.1 用户视角（验收线索）

| 场景 | 期望 |
|------|------|
| **Toggle 默认关**（若产品确认） | 不见右键项、不见头部 home；cwd 体感为「整库」；路径展示用 **`/`**。 |
| **Toggle 开 → 右键子文件夹** | Ribbon 与子标题（及 **所有 Tab**）尽快一致；再发一问时 **Claude/Codex** 在项目子目录下读配置；`/ ` 列表与运行时一致。 |
| **切换后** | 无需重启 Obsidian；若正在流式输出，被取消或给出明确Notice（对齐 env 变更策略）。 |

### 6.2 程序视角（不变量）

| 不变量 | 说明 |
|--------|------|
| **单一写入源** | 工作空间语义变更必须经过 §4.2 **唯一管线**，避免出现 **Storage 已是 B 而 Tab 仍是 A**。 |
| **缓存一致性** | **`ClaudeCommandCatalog.sdkCommands`** 与 **`getSdkCommands` 运行时注入**先后顺序明确；切换后 **`refresh` + invalidate dropdown cache**。 |
| **`resolveWorkspacePath` 合法性** | 非法目录仍回落 **`null`/根**，不抛未捕获异常打断 UI。 |

## 7. 已知边界与后置迭代

1. **`Cursor`** 子目录工作空间：**未**在本期代码路径内；若在统一产品体验上必须对齐，需 **`CursorChatRuntime.query`** 改用 Tab **`effectiveCwd`** 映射（并核对 CLI **`--workspace`** 语义）。
2. **`AgentManager`(vault 根)** vs **运行时 effectiveCwd**：若未来要求「本子目录的智能体配置文件」也完全分离，需在 **`agents` 读写路径**上做 scoped 拆分（超出本篇「Toggle + 刷新」最小闭环）。
3. **多窗格多个 `ClaudianView`**：§4 **「所有视图所有 Tab」** 必须遍历 **`getAllViews()`**（ **`main`** 已有范式）。
