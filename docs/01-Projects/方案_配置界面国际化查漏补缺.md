# 方案：配置界面国际化查漏补缺（中英）

> **依据**：《方案_国际化仅中英与智能体输入占位符.md》第二阶段、《差异清单_国际化与配置扫描.md》第四节 P4 等。  
> **日期**：2026-04-19  
> **范围**：设置页（通用 / Claude / Codex）及子模块（插件列表、Codex 技能与子代理、环境区块、安全模式下拉等）。

---

## 1. 问题归纳

在 **简体中文（`zh-CN`）** 下，配置界面仍出现大量英文，原因分三类：

| 类型 | 说明 | 典型表现 |
|------|------|----------|
| A. 中文包未翻译 | `zh-CN.json` 中键存在但值仍为英文 | `claudeSafeMode`、`customModels` 标题与说明 |
| B. 代码硬编码 | 组件未调用 `t()` | Codex 设置 Tab 全文、`PluginSettingsManager`、`CodexSkillSettings`、`CodexSubagentSettings` |
| C. 枚举直出 | 下拉用内部值作展示文案 | `acceptEdits` / `default`、`workspace-write` / `read-only`、推理摘要 `Detailed` 等 |

产品名（Claude、Codex、Claudian）、路径、模型 ID、环境变量键名、技术占位示例可保留英文；**面向用户的标签、说明、空状态、Notice、无障碍标签** 应走 `t()`。

## 3. 实施策略

1. **新增翻译键**：在 `en.json` / `zh-CN.json` 同步增加 `settings.codex.*`、`settings.claudeEnv.*`、`settings.pluginsUi.*`、`settings.codexSkills.*`、`settings.codexSubagents.*`、`settings.claudeSafeMode.option*`、`settings.codexSafeMode.option*`、`common.learnMore` 等；**同步扩展** `src/i18n/types.ts` 中 `TranslationKey`。  
2. **下拉选项**：`addOption` 的 **value** 保持与存储一致，**第二参数** 使用 `t()`。  
3. **校验 / Notice**：凡用户可见句子一律 `t()`，必要时使用占位符 `{name}`、`{message}`。  
4. **验收**：界面语言设为简体中文后，上述截图区域无残留英文说明句；英文界面仍为自然英文。
