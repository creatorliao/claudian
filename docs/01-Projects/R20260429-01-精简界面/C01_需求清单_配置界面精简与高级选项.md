# C01 需求清单 · 配置界面精简与高级选项

**项目目录**：`docs/01-Projects/R20260429-01-精简界面/`  
**创建日期**：2026-04-29  
**状态**：已记录（待方案确认与实施）

---

## 1. 背景与问题

当前本插件「设置 → 通用」等内容较多（自定义系统提示词、排除标签、媒体文件夹、Vim 映射、快捷键说明、环境变量等），对初次使用的用户认知负担大，容易无从下手。

---

## 2. 目标

在**不削减既有能力**的前提下，通过**默认收起高级项**，让初级用户只看到少量高频、易懂的选项；需要时再展开「高级选项」访问其余配置。

---

## 3. 功能需求

### 3.1 高级选项开关

| 编号 | 需求描述 |
|------|----------|
| R1 | 增加「高级选项」开关（Toggle），**默认关闭**。 |
| R2 | 开关关闭时：隐藏「对初级用户不友好」的配置区块（见 3.3）。 |
| R3 | 开关打开时：展示与当前版本等价的全量通用设置（保持现有行为与数据兼容）。 |

### 3.2 简易模式（高级选项关闭）下须保留并展示的项

以下项目**必须可见**，且**顺序**如下（自上而下）：

| 顺序 | 配置项（用户表述） | 说明 |
|------|-------------------|------|
| 1 | **用户名称**（希望如何称呼你） | 个性化称呼；须置于最前 |
| 2 | **智能体名称** | 对话界面左上角商标旁展示名称 |
| 3 | **聊天提供商** | 启用/选择与对话相关的提供商（与现有「提供商」开关语义一致） |
| 4 | **语言** | 界面语言 |
| 5 | **最大聊天数**（标签页数量上限） | **默认值应为 5**（与当前产品中若默认非 5，则新产品/新安装以 5 为准，见方案文档） |

### 3.3 简易模式下须隐藏的内容（归入「高级选项」展开后可见）

高级选项为**关闭**时，除 3.2 所列五项外，**通用页**内其余区块均不展示，包括但不限于：

- 显示类：标签栏位置、在主标签页打开、自动滚动等（若当前位于「显示」分组）
- 会话类：自动生成标题、标题所用模型等
- 内容类：自定义系统提示词、排除的标签、媒体文件夹
- 输入类：Vim 风格导航映射
- 快捷键说明区块（跳转 Obsidian 快捷键设置）
- 环境变量 / 共享环境 / 自定义上下文限制等相关 UI

> **说明**：顶部「通用 | Claude | Codex | …」等多 Tab 结构是否调整，以 `C03` 解决方案为准；本需求优先保证「通用」页内字段分层清晰。

---

## 4. 非功能需求

| 编号 | 说明 |
|------|------|
| N1 | 持久化：高级选项开/关状态应写入插件配置，重启 Obsidian 后保持。 |
| N2 | 可访问性：开关须有清晰文案，便于用户理解「展开后会出现更多专业项」。 |
| N3 | 现有用户已有配置不得因本次改版被静默清空；默认值调整策略见方案（如新安装默认最大聊天数为 5）。 |

---

## 5. 验收要点（草案）

- [ ] 默认进入设置时，「高级选项」为关闭，仅见 3.2 中五项（及必要的分组标题，若有）。
- [ ] 用户名称、智能体名称位于简易列表最上方。
- [ ] 打开「高级选项」后，通用页展现完整选项集，与功能描述一致。
- [ ] 新安装或文档约定场景下，最大聊天数默认值为 5（与代码/迁移策略一致）。
