# C08 问题与方案：长文自动撑开 + 外层/内层双缘拖拽

> 记录日期：2026-04-29  
> 前置：**C05 §12**、**C06/C07** 已落地；本文件为新增需求与实现决策。

---

## 1. 问题 1：长文本下输入区不「自动撑开」观感异常

### 1.1 现象

向 `textarea` 粘贴或输入大量文本时，**内层仍很快出现纵向滚动条**，而组合器（尤其用户已拉高）或内层卡片外侧仍有富裕高度，整体像「框变大了字还在小窗格里滚」，与直觉不符。

### 1.2 原因（实现侧）

`autoResizeTextarea`（`Tab.ts`）将 **`max-height`** 固定为  
`max(150px, 55% × .claudian-container 高度)`。用户通过拖拽把 **内层 `.claudian-input-wrapper` 可用高度** 拉到大于该上限时，**`max-height` 仍按视窗比例封顶**，textarea 无法占满内层槽位，内容被迫在 **较小的可视高度** 内滚动。

### 1.3 结论

在计算 textarea **有效上限** 时，除保留既有 **视窗比例安全上限** 外，引入 **内层槽位高度**：

- 测量 **`wrapper.clientHeight`** 减去 **除 textarea 外子节点高度**（context 行、toolbar 等）得到 `slotCap`。
- 取 **`max(globalCap, slotCap)`**，使「用户拉高组合器」后 textarea 可随 **内层卡片** 一起变高，减少 **外高内 Scroll** 的割裂感。
- 仍以 **`min(..., 视窗硬顶)`**（例如约 **0.92 × 视窗高**）防止极端布局失控（数值可与产品再收紧）。

## 2. 问题 2：外层顶缘与内层顶缘都要能拖拽

### 2.1 现象与期望

- **内层顶缘**（导航栏与内层带圆角卡片之间）已有隐形命中条，手感符合预期。
- 用户希望 **外层组合器顶缘**（侧栏主题下外卡片最上沿、紧贴队列/顶栏之上）也具备 **同等** 调整逻辑：**同一套** `min-height` / 持久化 / 钳制 / `autoResizeTextarea`。

### 2.2 方案

1. 在 **`.claudian-input-container` 内、队列行之前** 增加第二条 **`.claudian-composer-resize-handle`**（与内条 **同一 CSS 类**，视觉仍默认无线）。
2. **`ComposerResizeHandle`**：对 **两个 DOM 节点** 绑定 **同一套** `pointerdown` / document 级 `pointermove|up|cancel`；`activeDrag` 记录 **实际捕获指针的那个把手元素**，收尾时 **`releasePointerCapture`** 对应节点。
3. **无障碍**：仅 **内层** 把手保留完整 **`role="separator"` + `aria-*` + `tabIndex=0`**；**外层** 把手 **`role="presentation"` + `aria-hidden="true"` + `tabIndex=-1`**，避免读屏重复播报（键盘用户仍可聚焦内条）。

### 2.3 DOM 顺序（更新后）

**outer handle** → **队列行** → **导航行** → **inner handle** → **`.claudian-input-wrapper`**（context → textarea → toolbar）。

### 2.4 双击「外条与内条之间」区域重置高度（C9）

- **区域**：**队列行**（`.claudian-input-queue-row`）与 **导航行**（`.claudian-input-nav-row`）——位于 **outer handle 之下、inner handle 之上**，即内外拖拽条之间的整块非内层卡片区域。  
- **行为**：与 **双击任一条拖拽条** 相同：清除 `composerPreferredMinHeightPx`、落盘、去掉容器 `min-height` 内联样式并 `applyComposerMinHeightFromSettings` + `onAfterApply`。  
- **防误触**：双击目标若落在 **`button` / `a` / `input` / `textarea` / `select` / `[contenteditable="true"]` / `[role="button"]`** 等可交互子元素上，**不**执行重置（避免与队列 Steer、导航图标等冲突）。

### 2.5 工具栏贴内层卡片底边（需求 4）

当 `textarea` 的 **`max-height`**（含 C8 槽位/视窗钳制）小于内层 **flex** 分配高度时，列方向默认 **`flex-start`** 会把多余空白堆在 **`textarea` 与 `.claudian-input-toolbar` 之间**，发送区视觉上像「上浮」。**对策**：为 `.claudian-input-toolbar` 设 **`margin-top: auto`**，让剩余主空间由自动外边距吸收，工具栏始终贴 **`.claudian-input-wrapper` 底边**；`textarea` 仍 `flex: 1` 优先长高，占满时 `margin-top: auto` 为 0。

## 4. 验收要点

- [x] 长文：拉高组合器后 textarea 优先占满内层槽位再内部滚动（`max(globalCap, slotCap)` + 视窗软顶）。
