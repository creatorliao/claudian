# C03 解决方案（初稿）：通过拖拽调整输入框高度

## 1. 目标

在保留现有「随输入内容自动增高」能力的前提下，增加**用户主动拖拽输入区上边缘**以设定**偏好高度**，并**持久化**；默认仍为当前布局下的最小可用高度，且**禁止缩小到最小值以下**。

## 2. 仓库调查结论

### 2.1 DOM 结构（主对话）

`src/features/chat/tabs/Tab.ts` 中 `buildTabDOM` 构建顺序大致为：

- `.claudian-input-container` — 外层组合器（Obsidian 视图下还有卡片边框与阴影，见样式文件）。
- `.claudian-input-queue-row`（若有）。
- `.claudian-input-nav-row` — 顶栏（标签、快捷操作图标等）。
- `.claudian-input-wrapper` — **内层带边框容器**，内含 `.claudian-context-row` 与 `.claudian-input`（`textarea`）。

截图中「整输入区块上方的边框」对应 UX 上适合放置 **resize 命中条** 的位置：**紧贴 `.claudian-input-container` 顶部内侧或顶部边缘**，拖拽时改变下方整块 composer 的预留高度（视觉上等价于把「消息区 / 输入区」分界上移或下移）。

### 2.2 高度与自适应逻辑

| 位置 | 说明 |
|------|------|
| `src/features/chat/tabs/types.ts` | `TEXTAREA_MIN_MAX_HEIGHT = 150`、`TEXTAREA_MAX_HEIGHT_PERCENT = 0.55`，用于内容驱动的最大高度上限。 |
| `src/features/chat/tabs/Tab.ts` | `autoResizeTextarea(textarea)`：根据内容与 flex 分配高度比较，必要时给 `textarea` 设 `min-height`；并设置 `max-height` 为 `max(150px, 55% 视图高度)`。 |
| `src/style/components/input.css` | `.claudian-input-wrapper` `min-height: 140px`；`.claudian-input` `min-height: 60px`、`resize: none`。 |
| `src/style/features/obsidian-layout-claudian-view.css` | 侧栏视图下对外层 `.claudian-input-container` 的卡片化样式（圆角、阴影等）。 |

**结论**：当前高度由 **CSS 最小高度 + flex 分配 + JS 按内容追加 min-height** 共同决定；要实现「用户基准高度」，需要在「flex 给 textarea 的基础高度」或 **wrapper / 中间层**上引入**用户可控的下限**，并与 `autoResizeTextarea` **合并策略**，避免两处互相覆盖导致抖动。

### 2.3 持久化

- 全局设置类型：`src/core/types/settings.ts` 中 `ClaudianSettings`。
- 默认值：`src/app/settings/defaultSettings.ts` 中 `DEFAULT_CLAUDIAN_SETTINGS`。

新增字段建议挂在 **UI preferences** 一类（与 `enableAutoScroll`、`tabBarPosition` 同级语义），例如：`inputComposerUserMinHeightPx`（数值）或 `composerPreferredHeightPx`，加载后钳制在 `[绝对最小, 计算得到的最大]` 区间内。

## 3. 初步方案

### 3.1 交互设计

1. **拖拽条（Resize handle）**  
   - 在 `.claudian-input-container` **顶部**放置一条窄命中区（例如高度 4～6px，可含 1px 视觉线），`cursor: ns-resize`。  
   - `mousedown` 记录起始 Y 与起始高度；`mousemove`（挂在 `document` 上）按 **向上拖 → 增高 composer** 更新偏好高度；`mouseup`/`mouseleave` 结束并写回设置。  
   - 拖拽过程中可加 `user-select: none`、临时屏蔽文本选中，避免误选。

2. **最小高度**  
   - **程序最小值**：不小于当前设计下的组合最小高度（可与 `.claudian-input-wrapper` 的 `min-height: 140px` 及顶栏、toolbar 占位对齐，实施时以实测 DOM 总和为准，建议在代码里集中定义为常量）。  
   - **默认**：未保存或首次使用时，等价于「仅 CSS 默认」，即**不额外施加用户像素偏好**。

3. **最大高度**  
   - 与现有策略一致：不应超过 `autoResizeTextarea` 使用的上限（`max(150px, 55% · 视图高度)`），否则与「内容最大高度」语义冲突。拖拽上限可取 **同一公式** 或与 wrapper 总高度联动计算。

### 3.2 与 `autoResizeTextarea` 的配合（要点）

可选策略（实施阶段二选一并写测试）：

- **策略 A（推荐）**：用户偏好高度作为 **`textarea` 所在列 flex 区的基础 min-height**（或对 `.claudian-input-wrapper` 设 `min-height`，其中包含 nav/context/toolbar）。`autoResizeTextarea` 仍在内容变长时在「不低于用户 min」的前提下继续增高，直至触及原有 `max-height`。  
- **策略 B**：仅提高「空内容时的占位高度」，内容变短时不低于用户拖拽高度 —— 与 A 类似，需在 `input` 事件中重新应用用户下限。

无论哪种，拖拽结束时调用一次 **`autoResizeTextarea`**，避免视觉突变。

### 3.3 持久化与默认值

- 在 `ClaudianSettings` 增加可选数字字段；`defaultSettings` 不设或设为 `undefined`，表示使用纯 CSS 最小布局。  
- `loadSettings` 后对历史异常值（负数、过大、`NaN`）做钳制。

### 3.4 样式与主题

- 命中条样式放入 `src/style/components/input.css` 或 `obsidian-layout-claudian-view.css`，注意 **明暗主题**下边框可见性。  
- 避免影响 `.claudian-container--header-mode` 等已有布局变体（若有顶栏迁移，拖拽条仍依附输入组合器顶部）。

### 3.5 风险与边界

| 风险 | 缓解 |
|------|------|
| 窗格极窄时纵向空间不足 | 最小高度钳制 + 与消息区 flex `min-height: 0` 现有约束兼容；必要时拖拽上限随 `clientHeight` 动态计算。 |
| 触控设备无 hover | 初版可仅保证鼠标；后续可加粗命中区或长按模式（非必须）。 |
| 与 Obsidian 拆分「消息 / 输入」布局的未来改版冲突 | 拖拽逻辑绑定在「输入组合器容器」上，改版时迁移容器类名即可。 |
