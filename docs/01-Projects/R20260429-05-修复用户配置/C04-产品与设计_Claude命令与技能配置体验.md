# 产品与 UX 设计：Claude「命令与技能」配置（工作区 / 全局）

本文档在 **[C03-解决方案(初稿)](C03-解决方案(初稿)_Claude命令与技能来源范围.md)** 的功能约束之上，从**产品经理与用户体验**角度定义**信息架构、文案、渐进披露与视觉层级**，使实现结果与 **Claudian 现有 Claude 设置页**（`Setting` 标准行、`claudian-sp-*` 列表区、「显示更多」高级块）**一致、简单、可理解**。

---

## 1. 设计意图（为何这样设计）

### 1.1 降低认知负荷

- 用户目标通常是：**「对话里能有我想要的 / 命令」**，而不是学习 `settingSources`、磁盘路径或 Vault 与主目录的加载顺序。
- **一个主决策**替代「加载用户设置」与「命令从哪里来」两套独立控件，避免**自相矛盾**（开着一个、关着另一个）带来的困惑。

### 1.2 与现有习惯对齐

- Claude 页已有清晰分区：**安全**（标题）→ **命令与技能**（标题 + 说明段 + 列表容器）→ 子代理 → 插件 → **显示更多** → 模型 / 隐藏命令 / MCP / 环境 / CLI 等。
- 新能力应**落在「命令与技能」小节内**，复用现有 **`claudian-sp-settings-desc`**、**`claudian-slash-commands-container`**、列表 **`claudian-sp-header`** + **「+」** 按钮等样式，避免另起一套视觉语言。

### 1.3 默认友好，高级不打扰

- **默认**：「本库 + 电脑上共用的配置」（对应 C03 的 `vault-and-user-home`），与多数 Claude Code 用户心智一致。
- **少数**需要隔离库的用户主动改为「仅本库」即可；技术细节与风险提示**不在首屏堆砌**。

### 1.4 长列表不压垮设置页

- 工作区与全局合并后条数可能很大；**主界面只保留约 5 条摘要**，其余进入**弹窗「查看全部」**，防止整页滚动失控（与 C03 一致，此处固定为体验规范）。

### 1.5 不该看见的不默认展示

- **路径**（如 `~/.claude/...`）、**SDK 枚举**、**数据字段名**从主界面**收拢**到「了解详情」或高级区说明，减少「像在改配置文件」的压迫感。
- **运行时内置 / 仅探测用**的命令与 **设置页可管理的文件型命令/技能** 混排时：主列表**只展示可理解与可管理边界内**的条目；与现有 **「隐藏命令」**（高级区）分工明确——前者管**下拉噪音**，后者仍管**用户不想看到的命令名**。

## 2. 信息架构（页面上的位置）

### 2.1 「安全」区块的调整意图

原 **「加载用户 Claude 设置」** 开关若与「命令/技能范围」合并为**单一来源下拉框**，则**安全区不再重复**同一语义开关，避免两处描述不一致。

- **安全提醒**（权限、安全模式可能被用户侧规则影响）**不消失**：改为挂在**「本库 + 共用」**选项的说明文案中（见 §3.2），仍在用户做选择的**同一时刻**可见，而不是拆成第二个神秘开关。

### 2.2 「命令与技能」区块结构（自上而下）

与现有 `renderSlashSkillsSection` 顺序对齐，建议实现为：

1. **小节标题**：「命令与技能」（沿用 `settings.slashCommands.name`）。
2. **说明段**（一段 `setting-item-description`）：一句话说明**此处管理的是对话里 `/` 触发的自定义命令与技能**；保留「了解更多」链到官方 Skills 文档（与现有一致）。
3. **新增：标准 `Setting` 一行（下拉框）**  
   - **名称**：面向用户的短标题，例如 **「命令与技能的来源」**（实现时走 i18n）。  
   - **描述**：**一行通俗话** + 需要时的**安全/权限一句提示**（随当前选项变化可选，见 §3.2）。  
   - **控件**：两档下拉（与「安全模式」同一交互范式）。
4. **列表区域**（现有 `claudian-slash-commands-container`）：**摘要列表（≤5 条）** + **「查看全部（N）」** + **「+」** 添加（仅对可在本库创建的条目生效；与现有行为一致）。
5. **不**在小节首屏重复展示 `slashAssetScope`、`loadUserClaudeSettings` 等键名。

### 2.3 与「显示更多」的关系

- **来源范围**、**主列表 + 查看全部** 属于**高频、与日常使用强相关** → 留在**固定区**（不折叠进「显示更多」）。
- **隐藏命令**、**自定义模型**、**CLI 路径** 等保持现有**高级区**策略；**技术补充说明**若需要长文，可放高级区简短一行 + 外链，或折叠在「命令与技能」下 **「技术说明 ▼」**（可选，见 §4.3）。

## 3. 文案与选项命名（用户可见）

### 3.1 下拉选项（推荐中文）

| 内部值 | 主标签（Dropdown） | 设计说明 |
|--------|-------------------|----------|
| `vault-and-user-home` | **本库和本机共用** | 强调「同一台电脑上别的环境也用的那套」，避免背诵 `~/.claude`。 |
| `vault-only` | **仅本库** | 适合便携库、隔离试验；语气短、无否定堆叠。 |

（英文 locale 可对等为 *This vault + shared on this computer* / *This vault only*，避免直译生硬，由 i18n 同事润色。）

### 3.2 描述与安全提示（随选项切换）

- **仅本库**：描述聚焦**隔离**——例如「只使用当前库里的命令与技能，不使用本机用户目录下的配置。」用**不说路径**也能懂的话；可选灰字一句「与独立项目或多人共用机器时的隔离需求更匹配。」
- **本库和本机共用**：描述聚焦**一致性**——例如「与 Claude Code 在本机其他目录中的习惯一致，包含本机中的共用命令、技能与权限类设置。」  
  - **安全一句**单独成句或紧跟其后：「共用设置中的权限规则可能影响安全模式行为，请确认来源可信。」（与当前「加载用户设置」的警示等价，不弱化合规信息。）

不在主描述里写 `settings.json`、`settingSources` 字面。

### 3.3 模块级说明段（标题下第一段）

现有 copy 写死「Vault 级 .claude/commands…」容易在**全局合并**后显得过时。建议调整为**中性一句**：

- 示例：「管理对话中通过 `/` 使用的自定义命令与技能。」  
- 具体「存在本库文件夹里 / 与本机共用」**交给下一行「来源」下拉的描述**，避免三段重复。

（实施时更新 `settings.slashCommands.desc` 或拆成 desc + 简短补充键。）

## 4. 列表与「查看全部」交互

### 4.1 与现有列表头部一致

- 保持 **`claudian-sp-header`**：左侧标签可仍为「命令与技能」或改为「预览」类短词（若与标题重复则改为 **「已启用的预览」** 或仅图标 + 「+」——以不重复三层同标题为准）。
- 右侧：**「+」**（添加，主路径不变）；当 **N > 5** 时增加**次级文字按钮** **「查看全部（N）」**（样式与现有 `claudian-settings-action-btn` 一致，避免大屏按钮抢了「+」）。

### 4.2 预览行内容（每条）

- **主行**：`/名称` + 可选 **轻量角标**：「技能」/「命令」（沿用现有着色或 `claudian-slash-item-badge`）。
- **次行**：一行描述（截断）。
- **来源**：**不为**外行默认展示「Vault / user-home」英文；用**短中文** **「本库」** / **「本机共用」** 小字或标签，满足「我这条到底是哪边的」又不吓人。
- **操作**：本库可编辑删除；本机共用**隐藏删除、隐藏「编辑」主流程**，可保留 **「在资源管理器中打开」**（若实现成本高，首版可仅展示只读 + 说明「请到对应文件夹修改」——收在次要帮助里）。

### 4.3 弹窗「查看全部」

- **标题**：例如 **「全部命令与技能（N）」**，Modal 使用与现有 `SlashCommandModal` / 子代理 Modal 一致的 **`claudian-sp-modal`** 类。
- **正文**：可滚动列表，每行信息密度与预览一致；**编辑/删除**规则同 §4.2。
- **顶栏**：可选 **搜索框**（若 N 常大于 20，强烈建议；若首版工期紧，可列为二期）。
- **底栏**：关闭即可；无需「保存」，因编辑仍走单条 Modal。

### 4.4 空状态

- **0 条**：保持现有一类「空列表」短句；若来源为「仅本库」可提示「可在本库 `.claude` 下添加」——**可选**用 Obsidian 内链或弱化路径；避免首屏大段教程。

## 5. 刻意隐藏或弱化呈现的内容

| 内容 | 主界面 | 备注 |
|------|--------|------|
| `~/.claude/...` 路径 | 默认不出现 | 高级「环境 / 技术说明」或官网文档 |
| `settings.json`、`settingSources` | 不出现 | 工程师文档 / C03 |
| 字段名 `slashAssetScope` | 不出现 | 配置 JSON 与代码 |
| 「用户主目录」「Vault」术语 | 尽量不用 | 改用「本机共用」「本库」 |
| 全局条目的编辑入口 | 隐藏 | 防误改；可选「打开文件夹」 |
| 完整长列表 | 不铺在设置页 | 仅弹窗 |

## 6. 与 C03 的对应关系（交付分工）

| C03（技术） | C04（体验） |
|-------------|-------------|
| `slashAssetScope` 默认值 `vault-and-user-home` | 下拉默认 **「本库和本机共用」** |
| 迁移 `loadUserClaudeSettings` | UI 仅保留一个下拉；旧键迁移逻辑不变 |
| 合并列表 + `provenance` | 界面展示 **「本库 / 本机共用」** |
| 主列表 ~5 + Modal 全量 | §4 交互与视觉规范 |
| `listDropdownEntries` 与运行时一致 | 用户无感；异常时高级区简短排查说明即可 |

## 7. 验收（体验向）

- 新用户**不展开「显示更多」**即可：选来源、看到预览、打开全部列表、添加本库命令。
- 默认选项下，**无需理解路径或 JSON** 仍能完成常用配置。
- 切换为「仅本库」后，列表与描述**同步体现隔离**，且页面**长度不随条数爆炸**。
- 安全相关提示在选用「本库和本机共用」时**一定可见**，符合现有合规力度。
