# 验收标准与测试清单：工作目录与右键切换

> 配合 [01-PRD](./01-PRD_工作目录与右键切换.md) 与 [02-实现方案](./02-实现方案_代码事实与复现步骤.md) 使用。  
> **环境**：桌面端 Obsidian；Vault 内至少两层文件夹；安装本插件并可打开 Claudian 视图。

---

## 1. 功能验收（必须通过）

### 1.1 右键「设为工作空间」

| ID | 验收项 | 预期 |
|----|--------|------|
| A1 | 仅 **文件夹** 右键出现「设为工作空间」 | 文件右键无此项（可有「添加到上下文」） |
| A2 | 点击后 **Notice** 成功 | 无报错 |
| A3 | `data.json` 中写入 `workspace` | 值为该文件夹的 **Vault 相对路径**（与文件树 path 一致） |
| A4 | Vault 外或非法路径 | `invalidPath` 类 Notice，不写入 |
| A5 | **已打开 Tab 的 cwd 行为** | 在当前实现下：**不**因右键而自动改变已有 Tab 的 `tab.workspace`；**新建 Tab** 后新 Tab 的发送/工具路径应以新目录为 `effectiveCwd`（可通过在对话中让模型 `pwd` 或读相对路径验证，若环境允许） |
| A6 | Ribbon tooltip | 右键后 **Ribbon** 提示应反映**新持久化**路径（简称或 Vault 根文案） |

### 1.2 重置工作空间

| ID | 验收项 | 预期 |
|----|--------|------|
| B1 | 标题栏 **folder-x** 或命令 `Reset workspace` | `Notice` 已重置 |
| B2 | `data.json` 的 `workspace` | 为空字符串或等价于「未设置」 |
| B3 | 当前视图所有 Tab | `tab.workspace` 为 `null`，副标题显示 Vault 根文案 |
| B4 | 新建 Tab | 仍以 Vault 根为有效工作目录 |

### 1.3 新建 Tab 继承

| ID | 验收项 | 预期 |
|----|--------|------|
| C1 | 持久化指向 `proj/A`，关闭所有 Tab 后新建 Tab | 新 Tab `workspace` 解析为 `proj/A` 的绝对路径 |
| C2 | 持久化为空 | 新 Tab `workspace` 为 `null` |

### 1.3b 多页签与工作空间（不串台 / UI 一致性感知）

| ID | 验收项 | 预期 |
|----|--------|------|
| C3 | 开 Tab1 后右键将全局设为目录 B，再新建 Tab2 | Tab1 仍应为**原**快照（未因右键而变）；Tab2 应为 B（或解析结果） |
| C4 | Tab1、Tab2 分别在不同快照下，交替点击页签再发消息（或通过工具验证 cwd） | 每次发送应对应**当前激活页签**的 `effectiveCwd`，不应混用另一页签目录 |
| C5 | 右键改全局后立刻看 Ribbon 与当前页签副标题 | 允许**暂时不一致**（Ribbon=持久化，副标题=当前 Tab）；验收时记录为已知特性或产品后续优化项 |

### 1.4 路径失效

| ID | 验收项 | 预期 |
|----|--------|------|
| D1 | 持久化指向已删除文件夹 | `resolveWorkspacePath` 返回 `null`；行为上等价 Vault 根（发送仍应有有效 `effectiveCwd` = Vault 根） |

### 1.5 会话恢复（若启用 Tab 持久化）

| ID | 验收项 | 预期 |
|----|--------|------|
| E1 | Tab1 在子目录 A、Tab2 在子目录 B，重启 Obsidian | 恢复后两 Tab 各自 `workspace` 与关闭前一致（见 `tabManagerState`） |

### 1.6 内部上下文过滤

| ID | 验收项 | 预期 |
|----|--------|------|
| F1 | 工作空间为子文件夹 A，打开 A 内与 A 外两个笔记 | 文档胶囊仅出现 **A 内**文件（外置文件行为与实现一致，可能带 Notice） |

### 1.7 Bang Bash（若开启 `enableBangBash`）

| ID | 验收项 | 预期 |
|----|--------|------|
| G1 | 在子目录工作空间 Tab 执行 `!pwd` 或等价 | 输出目录应为该 Tab 有效工作目录（在实现允许的前提下） |

## 4. 回归范围（建议）

- 多 Tab 切换、关闭、持久化恢复。
- `ExternalContextSelector` 与「添加到上下文」右键（勿与工作空间混淆）。
- `InputController` 发送消息仍带正确 `queryOptions.effectiveCwd`（可打日志或断点）。
