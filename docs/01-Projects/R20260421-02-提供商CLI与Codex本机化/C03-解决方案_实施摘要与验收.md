# C03 解决方案：Codex 本机化与 Cursor/Codex CLI 路径检测增强

## 1. 背景与目标

- **Codex**：移除 Windows 下「本机 / WSL」安装方式切换，**仅在本机进程**启动 Codex CLI；删除 WSL 相关设置、启动、路径映射与测试，保证 macOS / Windows / Linux 本机路径一致可运行。
- **CLI 检测**：在 Obsidian 等 **GUI 短 PATH** 场景下，提升 **Cursor Agent** 与 **Codex CLI** 的自动发现率；**Claude** 已有 `getEnhancedPath` 体系，本次以 Cursor / Codex 为重点。
- **体验**：精简 Codex 设置页「CLI 路径」文案；必要时对齐通用页三提供商开关与 presence 检测。

## 2. 架构与行为变更

### 2.1 Codex 提供商

| 领域 | 变更要点 |
|------|-----------|
| 设置模型 | `CodexProviderSettings` 移除 `installationMethod`、`installationMethodsByHost`、`wslDistroOverride`、`wslDistroOverridesByHost` 等字段。 |
| 执行目标 | `CodexExecutionTarget` 仅 `host-native` / `native-windows`；删除 `wsl` 与 `distroName`。 |
| 解析与启动 | `CodexExecutionTargetResolver`、`CodexLaunchSpecBuilder`、`codexAppServerSupport` 等去掉 WSL 分支；仅 spawn 本机 `codex` 可执行文件。 |
| 路径映射 | `CodexPathMapper` 仅身份或本机规范化，删除 UNC `/mnt` 等 WSL 映射。 |
| 运行时上下文 | `CodexRuntimeContext` / `CodexChatRuntime` 删除 WSL 专用 transcript 根路径推导。 |
| 设置 UI | `CodexSettingsTab` 移除安装方式、WSL 发行版控件；文案与占位与平台一致。 |
| 通用页 | `providerCliPresence`：Codex 与其他提供商统一用 `resolveCodexCliPath` 判断，不再对 WSL 特判「视为已安装」。 |

### 2.2 Cursor Agent CLI 检测（`CursorBinaryLocator`）

- **既有**：`getEnhancedPath` + PATH 扫描多文件名；`%LOCALAPPDATA%\cursor-agent\versions\<build>\` / `~/.local/share/cursor-agent/versions/` 按构建时间选最新二进制。
- **新增**：**IDE 捆绑目录**（GUI PATH 常不含）：
  - macOS：`/Applications/Cursor.app/Contents/Resources/app/bin`、`~/Applications/Cursor.app/.../bin`（存在才纳入）
  - Windows：`%LOCALAPPDATA%\Programs\cursor\resources\app\bin`
- **顺序**：PATH → **IDE bundled bin** → **cursor-agent 官方 versions 根**。

**依据**：Cursor 官方 CLI 安装文档（curl / PowerShell）、与 VS Code 系产品在 `resources/app/bin` 下放命令行工具的常见布局。

### 2.3 Codex CLI 检测（`CodexBinaryLocator`）

- **既有**：在 `getEnhancedPath` 合并后的目录中查找 `codex` / `codex.exe` / `codex.cmd`。
- **新增**：`getCodexSupplementalSearchDirectories`，与上述目录 **去重合并** 后再查找：
  - `npm_config_prefix` / `NPM_CONFIG_PREFIX` / `NPM_PREFIX`（Windows 为 prefix 根目录，Unix 为 `prefix/bin`）
  - `PNPM_HOME`（若设置）
  - `~/.npm-global/bin`、`~/.yarn/bin`
  - Windows：`%LOCALAPPDATA%\pnpm`

**依据**：OpenAI Codex 文档（`npm i -g @openai/codex`、Homebrew）、常见自定义 npm prefix / pnpm 全局目录。

### 2.4 国际化

- `settings.codex.cliPath.descGeneric` / `descWindowsNative` 缩短，与 Claude 选项卡「自定义路径 + 留空自动检测」句式对齐。

### 2.5 其他工程与测试

- **单元测试**：`Codex*` 全链路去 WSL；`CodexChatRuntime`、`CodexSettingsTab`、`ClaudianSettingsStorage`、`providerCliPresence` 等同步；Cursor 增加 `getCursorIdeBundledAgentBinDirectories`（Windows 临时目录）用例；Codex 增加 `npm_config_prefix` 补充目录用例。
- **杂项**：`CursorBinaryLocator` / `CursorAuxCliRunner` 等与 ESLint 对齐的小修正（如 import 顺序、`subdirs` 赋值）。
