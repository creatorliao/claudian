## 1. 代码现状（调查结果）

### 1.1 侧栏：按钮与回调

| 部位 | 文件与要点 |
|------|------------|
| 斜杠快捷按钮 **当前位置** | `src/features/chat/ui/InputToolbar.ts` 内 `createInputToolbar()`：当存在 `callbacks.openSlashCommandPicker` 时创建 `.claudian-slash-picker-toolbar`，`setIcon(slashBtn, 'slash')`，点击调用回调。 |
| 与输入 `/` 同源 | `src/shared/components/SlashCommandDropdown.ts` 的 `openSlashPickerFromToolbar()`：聚焦 `inputEl`，必要时插入 `/`，再 `handleInputChange()`，与 `input` 事件触发的解析 **同一套** 逻辑。 |
| Tab 接线 | `src/features/chat/tabs/Tab.ts` 中 `initializeInputToolbar()` 将 `openSlashCommandPicker` 设为 `tab.ui.slashCommandDropdown?.openSlashPickerFromToolbar()`。 |

### 1.2 样式

- 输入条旁斜杠按钮：`src/style/toolbar/slash-picker-toolbar.css`（`.claudian-slash-picker-toolbar` 等）。
- 头部操作按钮：`src/style/components/header.css`（`.claudian-header-btn` 已有 16px 图标与 hover）。

### 1.3 内联编辑（范围外说明）

- `src/features/inline-edit/ui/InlineEditModal.ts` 使用 **独立** `SlashCommandDropdown` 与 `.claudian-inline-slash-picker-icon`，**不经** `InputToolbar`。本次侧栏改动 **不自动删除** 内联入口；若未来要求视觉统一，可再单独把内联图标改为 `zap`。

## 2. 推荐方案（实施后行为）

### 2.1 布局与 DOM

1. **在 `ClaudianView.buildNavRowContent()`** 中，于 `headerActionsContent` 内 **增加** 一个按钮节点（建议使用已有 **`claudian-header-btn`**，必要时再加专用 class 以便主题微调）。
2. **按钮顺序（建议）**：放在 **「新建标签」按钮左侧**（同一 flex 行内、靠操作组起始），从左到右形如：**闪电（斜杠列表）→ 新建标签 → 重置工作区 → 新会话 → 历史**……  
   - 若你希望闪电紧邻「+」右侧，可在确认阶段改序，实现成本相同。
3. **从 `createInputToolbar()` 中移除** 整块 `claudian-slash-picker-toolbar` 的创建逻辑，并 **删除** `ToolbarCallbacks.openSlashCommandPicker` 及 `Tab.ts` 中对应回调字段，避免侧栏 **双入口**。
4. **点击处理**：`this.getActiveTab()?.ui.slashCommandDropdown?.openSlashPickerFromToolbar()`（`ClaudianView` 已有 `getActiveTab()`）。无激活标签时静默不做事（与当前极少出现的边界一致）。

### 2.2 图标与可访问性

- **`setIcon(..., 'zap')`**：仓库内已有先例（如 `McpSettingsManager` 测通、`InputToolbar` 内其他控件），一般对应闪电符号；若某 Obsidian 版本缺失，在实机换用文档中登记的备用 Lucide 名。
- **`aria-label`**：可复用现有 `t('chat.ribbon.openSlashCommands')`，或新增 `chat.header.openSlashCommands` 以区分「头部按钮」与旧 ribbon 文案；**以翻译文件实际条目为准**，避免英文/中文遗漏。

### 2.3 样式

- **优先**仅用 `claudian-header-btn`，与「新建标签」视觉一致。
- 若需与旧 `.claudian-slash-picker-toolbar-icon` 完全对齐（圆角 hover 块），可将该类 **附加** 到新按钮，或把 hover 规则抽一条到 `header.css`；**避免** 再依赖 `slash-picker-toolbar.css` 的 margin（头部 flex 已有 `gap`）。

### 2.4 数据与状态（无新逻辑）

- **禁止**复制命令列表或自建 Popover；**必须**继续走 `SlashCommandDropdown`。
- Bang bash 激活时 `slashCommandDropdown.setEnabled(false)` 后，`openSlashPickerFromToolbar()` 首部即 return，行为保持现状。

## 4. 测试与验收建议

- **手动**：侧栏单标签/多标签切换后，点击闪电，下拉是否为 **当前标签** 的 Provider 命令与技能；选择一项后是否插入输入框且下拉关闭。
- **回归**：`tabBarPosition` 为 `input` 与 `header` 两种模式下按钮均出现且可点；bang bash 开启并激活时按钮不误触发下拉。
- **自动**：若有针对 `SlashCommandDropdown` 的单元测试，保持不变；视情况为 `openSlashPickerFromToolbar` 补充 mock（非必须，以现有测试风格为准）。
