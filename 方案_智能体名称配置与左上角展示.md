# 方案：智能体名称配置与左上角展示

## 目标

在设置中新增一项与「Claudian 应该如何称呼你？」（`userName`）同类的纯 UI 偏好：**智能体显示名称**。用户可自定义对话界面左上角（Logo 旁）展示的名称；配置需持久化到 `.claudian/claudian-settings.json`，与现有设置读写路径一致。

## 现状简述

- **用户称呼**：字段 `userName`，在 `ClaudianSettingTab` 的「内容」分组下通过 `Setting.addText` 编辑，`onChange` 中写入 `plugin.settings` 并 `saveSettings()`。
- **左上角文案**：`ClaudianView.buildHeader` 中 `titleTextEl` 写死为 `'Claudian'`（`h4.claudian-title-text`），与 Logo 同处 `claudian-title-slot`。

## 设计决策

| 项目 | 说明 |
|------|------|
| 字段名 | `agentName`（字符串），与 `userName` 并列，语义为「界面展示用智能体名称」，不写入系统提示词、不影响模型行为。 |
| 默认值 | 空字符串 `''`。 |
| 展示规则 | `agentName.trim()` 非空则用该字符串；否则回退为 **`Claudian`**，与当前产品默认一致。 |
| 设置 UI | 紧接在「如何称呼你」下一行：标题/说明/单行输入，交互与 `userName` 相同（`onChange` + `saveSettings`）。无需 `blur` 时重启服务（与系统提示词类设置不同）。 |
| 即时刷新 | 保存后对所有已打开的 `ClaudianView` 调用公开方法刷新标题文本，避免用户必须重开视图。 |
| 工作区标签文案 | `ClaudianView.getDisplayText()` 与左上角保持一致，便于侧栏/主标签标题与品牌一致（可选增强，本方案一并实现）。 |

## 实现清单

1. **类型与默认值**  
   - `ClaudianSettings` 增加 `agentName: string`。  
   - `DEFAULT_CLAUDIAN_SETTINGS` 中 `agentName: ''`。

2. **持久化**  
   - 与现有 `ClaudianSettingsStorage.load` 的 `{ ...getDefaults(), ...stored }` 合并逻辑兼容，旧配置文件无该键时自动使用默认值。

3. **设置页**  
   - `ClaudianSettings.ts`：在 `userName` 的 `Setting` 之后新增 `agentName` 的 `Setting.addText`。  
   - `onChange`：`this.plugin.settings.agentName = value` → `await this.plugin.saveSettings()` → `this.plugin.getAllViews().forEach((v) => v.syncAgentDisplayNameInHeader())`。

4. **聊天视图**  
   - `ClaudianView`：抽取解析后的展示名；`buildHeader` 创建 `titleTextEl` 后调用同步方法；实现 `syncAgentDisplayNameInHeader()`（供设置页调用）；`getDisplayText()` 使用同一解析逻辑。

5. **国际化**  
   - `i18n/types.ts` 增加 `settings.agentName.name` / `settings.agentName.desc`。  
   - 各 `locales/*.json` 在 `settings.userName` 后增加 `agentName` 条目（英文为基准，其余语言提供对应翻译）。

6. **测试与类型**  
   - 若存在手写完整 `ClaudianSettings` 字面量的单测，补充 `agentName: ''` 以满足类型检查。

## 验收要点

- 设置「内容」中可见新项，输入保存后 JSON 中出现 `agentName`。  
- 清空该项后左上角恢复为 `Claudian`。  
- 多标签、Header 模式下原有显示/隐藏 Logo+标题逻辑不变（仍由 `updateTabBarVisibility` 控制）。  
- 未打开聊天视图时仅持久化即可；打开后修改设置应立即看到标题变化。
