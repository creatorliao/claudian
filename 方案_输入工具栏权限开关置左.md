## 1. 需求摘要

- 工具栏中 **`PermissionToggle`**（`.claudian-permission-toggle`）整体移到**整行最左**，紧挨在 **模型列表**（`.claudian-model-selector`）**左侧**。
- **内部**：**拨杆在左，文案在右**；**PLAN** 状态仅展示文案时，视觉与 Safe/YOLO 行高、风格协调。
- **不改**：`SendStopButton` 仍在 `Tab.ts` 中于 `createInputToolbar` **之后**创建，保持**最右**。

---

## 2. 当前实现逻辑（调查结论）

### 2.1 工具栏容器与挂载点

- **文件**：`src/features/chat/tabs/Tab.ts` → `initializeInputToolbar`。
- **`claudian-input-toolbar`** 挂在 `dom.inputWrapper` 下；**`SendStopButton`** 在 **`createInputToolbar` 之后** `new SendStopButton(inputToolbar, ...)`，故在 DOM 中**最后**，通常在**最右**。

### 2.2 子控件创建顺序（DOM 从左到右）

- **文件**：`src/features/chat/ui/InputToolbar.ts` → `createInputToolbar`。
- 当前顺序：**Model → … → MCP → Permission**；故权限块在**模型右侧**。

### 2.3 工具栏布局

- **文件**：`src/style/components/input.css` → `.claudian-input-toolbar`：`display: flex`、`justify-content: flex-start`，子项**无** `order`，**兄弟顺序 = DOM 顺序**。

### 2.4 权限块样式（与「置左」强相关）

- **文件**：`src/style/toolbar/permission-toggle.css`。
- **现状**：`.claudian-permission-toggle` 使用 **`margin-left: auto`**。在横向 flex 里，这会把该块**推向行尾**（吃掉左侧剩余空间）。  
  **结论**：仅把 `PermissionToggle` 改成「第一个子节点」而**不删改** `margin-left: auto`，权限块**仍可能贴在右侧**，与目标相反。置左时必须**去掉** `margin-left: auto`，改为与左侧对齐的间距（例如 **`margin-right`** 与模型区隔开，或依赖工具栏全局 `gap`，见 §3.2）。

### 2.5 `PermissionToggle` 内部与 PLAN 模式

- **文件**：`src/features/chat/ui/InputToolbar.ts` → `PermissionToggle`。
- **现状**：`render()` 先建 **`.claudian-permission-label`**，再建 **`.claudian-toggle-switch`**，即 **[文案][拨杆]**。
- **PLAN**：当 `mode === plan` 且支持计划模式时，`toggleEl.style.display = 'none'`，仅 **`labelEl`** 显示 **PLAN**（或配置的 `planLabel`），并加 **`plan-active`**。  
  改为 **[拨杆][文案]** 后，隐藏拨杆时仅剩右侧文案；需用样式保证**不塌陷、不显得空半行**（§3.3）。

## 3. 综合实施方案（待你最终确认后执行）

### 3.1 工具栏 DOM 顺序：`PermissionToggle` 置首

- **文件**：`src/features/chat/ui/InputToolbar.ts` → `createInputToolbar`。
- **改法**：将 **`new PermissionToggle(parentEl, callbacks)`** 移到 **`new ModelSelector(...)` 之前**；其余组件顺序不变。
- **结果**：Permission → Model → Thinking → … → MCP →（`Tab.ts`）SendStop。

### 3.2 样式：左对齐 + 与模型间距

- **文件**：`src/style/toolbar/permission-toggle.css`。
- **必做**：删除或覆盖 **`margin-left: auto`**，使权限块随 flex 起点排在**最左**。
- **建议**：
  - 使用 **`margin-right: 8px`**（或与设计一致的数值）与 **`model-selector`** 拉开距离；**或**在 `.claudian-input-toolbar` 上统一设 **`gap`**（若引入 gap，需快速扫一眼各子项是否已有足够间距，避免双重留白）。
  - 将原 **`padding-left: 12px`** 调整为符合「贴左」的视觉（例如减小左侧 padding、保留右侧 padding），避免左侧空一大块。

### 3.3 内部顺序：拨杆在左、文案在右；PLAN 美观

- **文件**：`src/features/chat/ui/InputToolbar.ts` → `PermissionToggle.render()`。
- **改法**：先 **`createDiv` 拨杆**（`.claudian-toggle-switch`），再 **`createSpan` 文案**（`.claudian-permission-label`）；`updateDisplay`、点击监听逻辑不变（仍绑在 `toggleEl`）。
- **PLAN 模式**（仅文案可见）：
  - **逻辑**：保持现有 `toggleEl.style.display = 'none'` 与 `plan-active`。
  - **美观**（建议同时做，可按实现微调）：
    - 给 **容器**在 PLAN 时增加状态类（例如在 `updateDisplay` 里当 `plan` 分支时 `this.container.addClass('claudian-permission-toggle--plan')`，退出时 `removeClass`），便于写样式。
    - **样式**（`permission-toggle.css`）：对 **`--plan`** 可做**轻量徽章感**：略增 **`padding`**、**`border-radius`**、**`background`** 用 `var(--background-modifier-hover)` 或低透明度品牌色，**`font-size` / `letter-spacing`** 与现有 `.plan-active`（青绿字 + `font-weight: 600`）协调，使单独 **`PLAN`** 字样在工具栏最左**不显得单薄或错位**。
    - 保持 **`align-items: center`**，避免仅文案时垂直偏移。

### 3.4 测试与回归

- **命令**：`npm run test -- --testPathPatterns=InputToolbar`（及按需全量测试）。
- **说明**：现有用例多断言类名与文案；若存在依赖 **DOM 子节点顺序** 的测试，按失败信息微调。
- **人工**：在 **Safe / YOLO** 与 **PLAN** 下各看一眼最左对齐与间距。

### 3.5 备选（仍不推荐）

- 仅用 CSS **`order`** 调整兄弟顺序而不改 `createInputToolbar`：**仍必须**处理 **`margin-left: auto`**，否则达不到「整块最左」；维护性不如直接调整构造顺序 + 样式。
