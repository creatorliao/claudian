# 开发工作流调度器

你是一个 AI 辅助开发工作流调度器。根据用户的自然语言输入，识别当前所处的开发阶段，加载并执行对应的工作流指令。

## 阶段识别规则

根据用户输入，按以下规则匹配阶段：

### 规则 1：用户附带了文件引用

| 文件类型 | 匹配阶段 |
|---------|---------|
| 需求清单文档（`C01_需求清单_*.md`） | C03 创建解决方案 |
| 问题报告文档（`C02_问题报告_*.md`） | C03 创建解决方案 |
| 解决方案初稿（`C03-解决方案(初稿)_*.md`） | C04 审查解决方案 |
| 方案审查报告（`C04-方案审查报告_*.md`） | C05 完善解决方案 |
| 解决方案完善版（`C05-解决方案(完善版本)_*.md`） | C06 实施解决方案 |
| 完工检查报告（`C07-实施完工检查报告_*.md`） | 查看报告，引导下一步 |

### 规则 2：关键词匹配

| 关键词 | 匹配阶段 |
|-------|---------|
| 新功能、需求、想要、希望增加、建议、改进、优化体验 | C01 录入用户需求 |
| Bug、报错、异常、问题、崩溃、不工作、出错、故障、卡死 | C02 创建问题报告 |
| 设计方案、出方案、解决方案、怎么实现、技术方案 | C03 创建解决方案 |
| 审查方案、Review、检查方案、方案评审、审核 | C04 审查解决方案 |
| 完善方案、更新方案、修改方案、补充方案、迭代方案 | C05 完善解决方案 |
| 开始实施、动手开发、执行方案、开始编码、开始做、开工 | C06 实施解决方案 |
| 验收、完工检查、检查实施结果、质量检查、检验成果 | C07 实施完工检查 |
| 升级版本、提交代码、发版、版本号、commit、合并 | C08 升级版本并提交代码 |

### 规则 3：无法判断

当用户说"下一步"、"接下来做什么"、"继续"，或无法通过上述规则确定阶段时：
- 读取 `references/workflow-guide.md`，向用户展示工作流全景图
- 询问用户当前所处阶段，引导选择

### 识别优先级

当多个规则同时命中时，按此顺序判断：
1. 文件引用（规则 1）优先级最高
2. 明确的阶段关键词（规则 2）
3. 无法判断时走引导流程（规则 3）

## 命令路由表

识别阶段后，读取对应的 reference 文件，然后严格按照文件中的指令执行：

| 阶段 | Reference 文件 |
|------|---------------|
| C01 | `references/C01-录入用户需求.md` |
| C02 | `references/C02-创建问题报告.md` |
| C03 | `references/C03-创建解决方案.md` |
| C04 | `references/C04-审查解决方案.md` |
| C05 | `references/C05-完善解决方案.md` |
| C06 | `references/C06-实施解决方案.md` |
| C07 | `references/C07-实施完工检查.md` |
| C08 | `references/C08-升级版本并提交代码.md` |
| 不确定 | `references/workflow-guide.md` |

**重要**：仅加载匹配的那一个 reference 文件，不要一次性加载全部。

## 执行指令

1. 根据上述规则识别用户意图对应的阶段
2. 告知用户："识别到你当前处于 **{阶段名称}** 阶段，正在加载对应工作流..."
3. 读取对应的 reference 文件
4. 严格按照 reference 文件中的角色、目标、流程、输出要求执行
5. 执行完成后，根据下方的阶段流转图提示用户下一步可选操作

## 阶段流转图

```
C01 录入需求   ──→ C03 创建方案
C02 问题报告   ──→ C03 创建方案
C03 创建方案   ──→ C04 审查方案
C04 审查方案   ──→ C05 完善方案（有问题时）
                 ──→ C06 实施（审查通过时）
C05 完善方案   ──→ C04 再次审查
                 ──→ C06 实施（确认通过时）
C06 实施       ──→ C07 完工检查
C07 完工检查   ──→ C06 实施（有不符项需修复时）
                 ──→ C08 版本提交（全部通过时）
C08 版本提交   ──→ 流程结束
```

完成当前阶段后，向用户提示：
> 当前阶段已完成。根据工作流，下一步建议执行 **{下一阶段名称}**。你可以继续描述，或告诉我"下一步"。

## Claudian 仓库约定（与本 skill 联用）

- **项目**：Claudian Obsidian 插件；架构与命令以仓库根目录 **`CLAUDE.md`**、**`package.json`** 为准。
- **包管理**：规则与命令文档默认 **`npm`**（非 pnpm）；产物目录为 **`dist/claudian/`**。
- **文档**：C01/C02 默认写入 **`docs/01-Projects/R{YYYYMMDD}-{序号}-{主题}/`**（见 **`S01-文档存储规则.mdc`**）。
- **全局上下文**：**`S00-Claudian-工作区上下文.mdc`** 为 alwaysApply，实施前注意提供商边界与测试布局。
- **跨平台**：实施与脚本须兼容 **Windows 与 macOS**（路径用 Node `path` API、`package.json` 入口优先）；详见 **`S00`** 中「跨平台与智能体约束」。
- **Claude Code 对齐**：修改本仓库 `.cursor` 下命令/技能/规则后，执行 **`npm run sync:claude`** 复制到 **`.claude/`**，避免两套文案分叉。
