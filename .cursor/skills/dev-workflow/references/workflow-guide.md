## 各阶段简述

| 阶段 | 编号 | 说明 | 适用场景 |
|------|------|------|---------|
| 需求输入 | C01 | 录入新功能需求 | 你有一个新想法或改进建议 |
| 需求输入 | C02 | 创建问题报告 | 你发现了一个 Bug 或异常 |
| 方案设计 | C03 | 创建解决方案 | 需求已明确，需要技术方案 |
| 方案审查 | C04 | 审查解决方案 | 方案已出，需要审查完整性 |
| 方案完善 | C05 | 完善解决方案 | 审查发现问题，需要修正 |
| 编码实施 | C06 | 实施解决方案 | 方案确认，开始编码 |
| 质量验收 | C07 | 实施完工检查 | 编码完成，验收检查 |
| 版本发布 | C08 | 升级版本并提交 | 验收通过，准备发版 |

## 如何选择？

- **"我有个新想法/需求"** → C01
- **"我发现了一个 Bug"** → C02
- **"需求已经记录好了，帮我出方案"** → C03
- **"方案出来了，帮我审查一下"** → C04
- **"审查有反馈，帮我更新方案"** → C05
- **"方案确认了，开始实施"** → C06
- **"实施完了，帮我检查"** → C07
- **"检查通过，升级版本提交"** → C08

## Claudian 仓库提示

- 架构与脚本：根目录 **`CLAUDE.md`**、**`package.json`**；构建产物 **`dist/claudian/`**；包管理默认 **`npm`**。
- 工作流文档默认落在 **`docs/01-Projects/`** 下 `R{日期}-{序号}-{主题}/` 目录（见 **`S01-文档存储规则.mdc`**）。
