# 方案：国际化精简为英文/简体中文，及智能体输入区占位符策略

> **状态（2026-04-19 更新）**：**第一阶段（语言清理 + 默认语言）已在代码库落地**；第二阶段（Ribbon/占位符等硬编码全面 `t()`）仍按本文档与《差异清单_国际化与配置扫描.md》排期。  
> **追溯**：已拍板事项见下文 **「8. 已执行决策记录（追溯）」**。

---

## 1. 目标

1. **语言范围**：产品侧仅维护 **English（`en`）** 与 **简体中文（`zh-CN`）** 两种界面语言。  
2. **智能体输入区**：**不应**使用类似 *「How can I help you today?」* 这类 **助手寒暄式、对话开场白式** 的占位符（差异清单 **R1**）。**已落地（2026-04-19）**：`Tab.ts` 主 `textarea` **不设置** `placeholder`，见 **§8.5**。  
3. **Ribbon / 设置 / 提示**：在语言精简完成后，按《差异清单_国际化与配置扫描.md》等文档逐项将硬编码改为 `t()`，并只维护 **en.json** 与 **zh-CN.json**。

---

## 2. 智能体输入区占位符（原则，须在方案中明示）

| 项目 | 要求 |
|------|------|
| **禁止** | 默认使用英文寒暄句作为占位符（例如 *How can I help you today?*、*What would you like to do?* 等同类话术）。 |
| **已选方向（主输入框）** | **A. 无占位符**：主聊天 `textarea` 不出现任何默认占位文案（**已实施**）。**B/C** 仍可用于其它入口，若未来需要短提示须走 `t()`。 |
| **指令模式 `#`** | `InstructionModeManager` 中 `# Save in custom system prompt` 等同类文案，应与上述原则一致：若保留提示，须 i18n 且避免「助手自我介绍」风格。 |

**说明**：主输入框 **R1** 已按 **§2 选项 A** 改代码；**`#` 模式**等仍待第二阶段（见 **§8.4**）。

## 3. 第一阶段：清理除简体中文、英文外的所有语言（确认后第一时间执行）

### 3.1 意图

- 减少维护面：删除 **zh-TW、ja、ko、de、fr、es、ru、pt** 等语言包与类型、注册表中的引用。  
- 避免用户选择到「已不再维护」的语言。  
- **注意**：若用户历史设置里存了将被删除的 `locale`，加载时需做兼容。**已实现规则**见 **§8.2**（保留 `en`；其余规范为 **`zh-CN`** 并写回设置）。

### 3.2 预期涉及位置（实施 checklist，非此刻修改）

| 类别 | 位置（示例） |
|------|----------------|
| 语言包文件 | `src/i18n/locales/` 下除 `en.json`、`zh-CN.json` 外的 JSON |
| 聚合加载 | `src/i18n/i18n.ts` 中对多语言的 `import` 与 `translations` 映射 |
| 类型 | `src/i18n/types.ts` 中 `Locale` 联合类型 |
| 常量与设置 UI | `src/i18n/constants.ts` 中 `SUPPORTED_LOCALES` |
| 文档 | `CLAUDE.md` 等写明「10 locales」处需改为仅 **2** 种 |
| 测试 | 引用 `ja`、`zh-TW` 等 locale 的用例需改为 `en` / `zh-CN` 或删除 |

### 3.3 清理完成标准（验收）

- 代码库中 **不存在** 对已删除 locale 的编译期引用。  
- 设置页语言下拉 **仅** English、简体中文。  
- 默认与迁移策略明确（**已实现**：见 **§8**；与早期草案「一律回退 `en`」不同，产品选择 **默认简体中文 + 过时 locale 归并 `zh-CN`**）。

## 4. 第二阶段：完善 en / zh-CN 国际化（在第一阶段完成后）

- 依据《差异清单_国际化与配置扫描.md》中的 **Ribbon（R1–R13）**、侧边栏、设置页、Notice 等条目，将硬编码字符串迁入 **`en.json` / `zh-CN.json`**，并在组件中调用 `t()`。  
- **智能体输入占位符**：按第 2 节选定方案实现；若使用文案，**两语言文件必须同时补齐**，且 **禁止** 恢复寒暄式默认英文占位。

### 4.1 与「智能体显示名称（`agentName`）」的衔接

- 设置项 **智能体显示名称** 已使用 `t('settings.agentName.name')` / `t('settings.agentName.desc')`（见《方案_智能体名称配置与左上角展示.md》）。  
- **第一阶段**删除其它语言包后，**仅需**在 **`en.json`、`zh-CN.json`** 中保留 `settings.agentName` 段，无需再为 zh-TW / ja 等维护该键。  
- **默认回退品牌名**：聊天壳左上角在 `agentName` 为空时显示 **`Claudian`**（`ClaudianView.getResolvedAgentDisplayName` 内硬编码）。若希望简体中文界面下默认标签也为中文，属**可选增强**：新增翻译键并在解析函数中在未配置 `agentName` 时使用 `t(...)`（需避免在 i18n 未初始化路径调用；实施时单独设计）。该条已记入《差异清单_国际化与配置扫描.md》**第九节 N1**。

## 5. 与差异清单 R1 的对应关系

- **R1** 指出的问题不仅是「未走 i18n」，在本方案中进一步规定：**即使做了 i18n，也不应采用「How can I help you today?」这类占位语义**；最终占位符形态以第 2 节为准。  
- 实施时应在任务单中同时关闭 **「硬编码」** 与 **「语义不当」** 两类问题。

## 6. 风险与依赖

- **用户既有配置**：`locale` 为将被删除语言时需迁移，避免加载失败或静默错误。  
- **贡献者与文档**：对外说明仅支持中英，避免 PR 再次引入多语言文件。  
- **Codex / Claude 提供商英文 label**：Ribbon 上模型名、YOLO/Safe 等可能仍来自 `*ChatUIConfig`；若需中文界面，可能需额外一层「配置 key → t()」设计（可在第二阶段子任务中拆分）。

### 8.1 语言范围与资源

| 决策 | 说明 |
|------|------|
| 仅保留两套翻译 | 语言包只保留 **`src/i18n/locales/en.json`**、**`zh-CN.json`**；已删除 **zh-TW、ja、ko、de、fr、es、ru、pt** 对应 JSON。 |

### 8.2 默认语言与历史配置迁移

| 决策 | 说明 |
|------|------|
| **新建默认语言** | `DEFAULT_CLAUDIAN_SETTINGS.locale`、**`i18n/constants.ts` 的 `DEFAULT_LOCALE`**、**`i18n.ts` 模块初始 `currentLocale`** 均为 **`zh-CN`**，保证未选过语言的新装即简体中文。 |
| **翻译键缺失时** | 当前界面为 `zh-CN` 时，若某键在中文包中缺失，**仍回退到英文词条**（实现中为 **`FALLBACK_LOCALE = 'en'`**），保证键集合以英文为完整基准。 |
| **磁盘上旧 `locale` 值** | 导出 **`normalizeClaudianLocale`**：`en`、`zh-CN` 原样保留；**任何其它字符串（含已移除语言代码）一律规范为 `zh-CN`**。在 **`main.ts` `loadSettings`** 合并设置后调用；若发生规范化，置 **`didNormalizeLocale`** 并随其它规范化逻辑 **`saveSettings`**，写回 `.claudian/claudian-settings.json`。 |
| **与 §3.1 草案的差异** | 草案仅要求「回退到 en 或 zh-CN 之一」；**产品拍板**：默认与迁移目标以 **简体中文** 为主，**仅显式 `en` 维持英语界面**。 |

### 8.5 主聊天输入框占位符（R1，已执行）

| 决策 | 说明 |
|------|------|
| **产品要求** | 用户 **不应** 在主输入区看到 *How can I help you today?* 或同类寒暄占位语；**默认无任何占位内容**。 |
| **实现** | `src/features/chat/tabs/Tab.ts` → `buildTabDOM`：创建 `textarea` 时 **省略** `placeholder` 属性（不再写死英文句，也无需 `t()`）。 |
| **连带行为** | `InstructionModeManager` / `BangBashModeManager` 仍用 `inputEl.placeholder` 保存/恢复「进入模式前」的占位符；主路径下该值为空，恢复后仍无占位。 |

### 8.6 空会话欢迎语（`getGreeting`，已执行）

| 决策 | 说明 |
|------|------|
| **原因** | `ConversationController.getGreeting()` 原在 **英文池** 中随机返回整句（含 *You are absolutely right!* 等），界面为 **简体中文** 时仍会出现英文。 |
| **实现** | 问候语全部迁入 **`chat.welcome.*`**（`en.json` / `zh-CN.json`），按 **`userName`** 使用带 `{name}` 与不带名字的成对键；逻辑见 **`ConversationController.ts`**。 |
