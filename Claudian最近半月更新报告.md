## 2. 2.0.0：架构级变化

- **新增 / 重大变更**
  - **多 Provider 架构**，并加入 **Codex 运行时（runtime）支持**（发布说明表述为 *multi-provider architecture with Codex runtime support*）。

---

## 3. 2.0.1：设置体验

- **调整（偏重构/体验）**
  - **设置（Settings）**：对设置标签页进行 **重新分组与排序**，以提升一致性与可浏览性。

---

## 4. 2.0.2：兼容性与小功能

### 4.1 修复

- **Windows 兼容性**：将 `aux` 相关目录重命名为 `auxiliary`，避免 Windows 对保留名路径的限制问题（#457）。
- **侧栏焦点**：在侧栏获得焦点时 **保持笔记选中高亮**（#454）。
- **路径解析**：对 wrapped adapters 使用 **共享的 vault 路径查找** 逻辑（#452）。

### 4.2 新增 / 改进

- **会话历史**：为对话历史增加 **新建标签页相关操作**（#453）。

---

## 5. 2.0.3：稳定性与交互修复（内容最多）

### 5.1 修复

- **聊天与链接**：恢复聊天中 **可见的 wiki 链接** 展示（#511）。
- **Claude 流**：合并/整理 **Claude 流的去重（dedup）** 逻辑（#510）。
- **标签与视图**：在创建新标签前先 **打开视图**，避免时序问题（#507）。
- **工具输入**：合并 **增量工具输入（incremental tool input）** 的更新处理（#508）。
- **路由**：在空白路由场景下 **尊重已启用的 Provider 配置**（#506）。
- **Codex**：处理 Codex app-server 初始化响应中 **缺失 `codexHome`** 的情况（#464）。
- **Hook**：前台任务完成后 **停止 hook 循环**（#502）。
- **会话保存**：避免在 **会话保存之前** 进行标签同步，降低竞态/错乱风险（#491）。
- **UI**：去除标签角标悬停时的 **重复 tooltip**（#467）。
- **平台**：在非 Windows 上 **隐藏 Codex 的 Windows 专用设置**（#489）。

### 5.2 重构

- **技能发现**：用 **SDK 探测 + 缓存** 模式替代原先基于 **文件扫描（file-scan）** 的技能发现方式。

---

## 6. 半月内汇总视角

### 6.1 修复类（问题与风险）

主要集中在：**多标签/视图与保存时序**、**流式输出去重**、**工具输入增量合并**、**Codex 初始化与平台相关设置**、**Windows 路径保留名**、**会话路由与 Provider 启用状态**、以及若干 **UI/交互细节**（tooltip、wiki 链接可见性、侧栏高亮等）。

### 6.2 新增功能 / 能力

- **2.0.0**：多 Provider + **Codex 运行时**（架构层面能力扩展）。
- **2.0.2**：会话历史中 **新建标签相关操作**。

### 6.3 体验与工程改进

- **2.0.1**：设置页信息架构调整。
- **2.0.3**：技能发现机制改为 **SDK 探测 + 缓存**（通常有利于性能与一致性，并减少全盘扫描）。
