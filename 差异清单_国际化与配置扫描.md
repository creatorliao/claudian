## 九、智能体显示名称（`agentName`）与左上角展示

> 需求与验收见《方案_智能体名称配置与左上角展示.md》。以下为扫描结论：**与 i18n 清单的交叉项**。

### 9.1 已实现（与方案一致）

| 项 | 说明 |
|----|------|
| 设置项 | `ClaudianSettings.ts`「内容」分组下，`userName` 之后：`settings.agentName.name` / `settings.agentName.desc`，`onChange` 写 `plugin.settings.agentName` 并 `saveSettings()`，再对 `getAllViews()` 调用 `syncAgentDisplayNameInHeader()`。 |
| 类型与默认 | `core/types/settings.ts` 中 `agentName: string`；`defaultSettings.ts` 中 `agentName: ''`。 |
| 视图 | `ClaudianView.buildHeader` 中标题初始为 `getResolvedAgentDisplayName()`；`getDisplayText()` 与之同源（工作区标签等）。 |
| 翻译键 | `i18n/types.ts` 含 `settings.agentName.name` / `settings.agentName.desc`；`en.json` / `zh-CN.json`（及其余现用语言包）含对应条目。 |

### 9.2 差异与后续（排期参考）

| 序号 | 类型 | 说明 |
|------|------|------|
| N1 | 默认品牌名硬编码 | `getResolvedAgentDisplayName()` 在 `agentName` 为空时回退为 **`'Claudian'`**（英文字符串，**未走 `t()`**）。中文界面下默认标签仍为英文品牌名；若需默认显示本地化文案，需产品决策并单独加键（例如 `chat.brand.defaultName`）。 |
| N2 | 多语言包维护面 | 当前 `agentName` 已写入 **10 个** `locales/*.json`。与《方案_国际化仅中英与智能体输入占位符.md》一致：执行 **语言精简** 后，仅需保留 **en / zh-CN** 中该段翻译，其余语言文件删除即不再维护。 |
| N3 | Header 可见性 | `updateTabBarVisibility`：≥2 标签且 **header** 模式下隐藏 Logo + 标题（方案已述）；与 i18n 无关。 |

## 七、Ribbon 专项：建议纳入 `t()` 的键分组（实施时参考）

便于后续只维护 **en / zh-CN**：

1. **`chat.ribbon.placeholder`**：输入框默认占位（R1）。  
2. **`chat.ribbon.instructionPlaceholder`**：# 模式（R2）。  
3. **`chat.ribbon.effortLabel` / `chat.ribbon.thinkingLabel`**（R3–R4）。  
4. **`chat.ribbon.fallbackUnknownModel` / `chat.ribbon.fallbackEffort` / `chat.ribbon.fallbackOff`**（R5）。  
5. **`chat.ribbon.tokenTooltip` / `chat.ribbon.disabled`**（R6）。  
6. **权限 / Fast**：要么继续由 `ProviderChatUIConfig` 返回 key 再由 UI `t()`，要么为 Claude/Codex 各加 locale 映射（R7–R8）。  
7. **`chat.ribbon.contextMeter.tooltip`**、`chat.ribbon.contextMeter.approachingLimit`（R9）。  
8. **外部上下文 / MCP** 整块中英文（R10–R11）。  
9. **模型列表**：`label`/`description` 是否翻译（产品需决策：模型名常保留英文，**描述**可翻译）（R12）。  
10. **`chat.ribbon.send` / `chat.ribbon.stop`**：`aria-label` + 可选 `title`（R13）。
