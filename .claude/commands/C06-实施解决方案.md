## 实施步骤

1. **分支**：若改动较大，新建分支；小修可直接在当前分支（按团队习惯）。
2. **理解方案**：对照 C05（或已确认的 C03）与 `CLAUDE.md`，明确涉及模块与提供商边界。
3. **分步实现与自测**：按验收标准分阶段完成；新行为或 Bug 修复优先补测试（`tests/` 镜像 `src/`）。
4. **静态检查**：消除 ESLint 问题；类型检查通过。
5. **构建**：执行 `npm run build`，确认 `dist/claudian/` 产物正确；开发模式可用 `npm run dev` 本地验证。
6. **版本与提交**：仅在用户要求发版时执行：按 **S03** 使用 `npm version`、更新 `CHANGELOG.md`、再 `npm run build`，然后 `git add` / `git commit`；默认不 `git push`。
7. **合并**：按团队流程合并默认分支。
