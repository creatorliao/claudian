# C06 实施解决方案

## 项目规则

执行本指令时请参考：`.cursor/rules/S02-构建系统规则.mdc`、`.cursor/rules/S03-版本发布规则.mdc`、`.cursor/rules/S00-Claudian-工作区上下文.mdc`；架构见 `CLAUDE.md`。

## 项目配置（Claudian）

- 包管理：**`npm`**
- 推荐门禁：`npm run typecheck && npm run lint && npm run test && npm run build`
- 生产产物：**`dist/claudian/`**

## 实施步骤

1) 大改动用新分支；小修按团队习惯。

2) 理解已确认方案（C05 或 C03）与 `CLAUDE.md`，明确 `core` / `providers` / `features` 边界。

3) 分阶段实现，按验收标准自检；新行为或修复优先写测试（`tests/` 镜像 `src/`）。

4) 消除 lint/类型错误。

5) **`npm run build`** 验证生产产物；日常可用 **`npm run dev`**。

6) 仅在用户要求发版时：按 S03 执行 **`npm version`**、更新 **`CHANGELOG.md`**、再 **`npm run build`**，然后 `git add` / `git commit`；默认不 `git push`。

7) 按流程合并主分支。

## 指令

严格按已确认方案实施；发现方案与代码严重冲突时先与用户沟通。
